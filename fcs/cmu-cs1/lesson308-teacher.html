Teacher Lesson Plan

Lesson 308: 2.1 Functions + 2.2 Mouse Events

Lesson Overview

Course: Fundamentals of Computer Science
Lesson Code: 308
Lesson Title: 2.1 Functions + 2.2 Mouse Events
Class Length: 90 minutes
CMU Unit: Unit 2
CMU Sections: 2.1, 2.2

This lesson introduces students to two foundational programming concepts: functions and mouse events. Students learn how to organize code into reusable functions and make programs interactive by responding to mouse actions. This marks a major shift from static drawings to dynamic, interactive programs. The lesson uses direct instruction followed by guided CMU CS Academy practice.

Objectives

Students will be able to:

  • Define and call custom functions to organize code.
  • Explain why functions help make code more readable and reusable.
  • Use onMousePress and onMouseMove to create interactive programs.
  • Understand how event handlers respond to user actions.
  • Combine functions and mouse events to build simple interactive graphics.

Materials & Setup

  • Student computers with internet access
  • CMU CS Academy login credentials
  • Student note-taking materials
  • Projector or teacher computer for live demonstrations
  • Access to CMU CS Academy Unit 2.1 and 2.2 activities

Teacher Preparation:

  • Review Unit 2.1 and 2.2 content in CMU CS Academy
  • Prepare simple function and mouse event examples to demonstrate
  • Test all code examples in advance
  • Have a plan for students who finish early

Teacher Moves (Scripted Talking Points + Demo Steps)

Hook and Introduction (5 minutes)

Talking Points

"Up until now, your programs have been static—they draw something and that's it. Today we're going to make programs that respond to you."

"We're also going to learn how to organize code so it's easier to read and reuse. This is one of the most important skills in programming."

"By the end of today, you'll be able to create programs that react when you move your mouse or click on the screen."

Direct Instruction: Functions (20 minutes)

1. What is a Function?

Explain:

"A function is a named block of code that performs a specific task. Instead of writing the same code over and over, you write it once in a function and call it whenever you need it."

Live Demo: Creating a Function

Write on the projector:

def drawStar():
    Star(200, 200, 50, 5, fill='gold')

Explain:

  • def means "define a function"
  • drawStar is the function name
  • The parentheses () are required
  • The colon : starts the function body
  • Everything indented belongs to the function

2. Calling a Function

Add to the demo:

drawStar()  # This runs the function

Key Point: "Defining a function doesn't run it—you have to call it by writing its name with parentheses."

3. Why Use Functions?

Discuss:

  • Organization: Keep code clean and readable
  • Reusability: Use the same code multiple times
  • Maintainability: Fix bugs in one place instead of many

Practice Example Together

Guide students through creating a function that draws a simple house, then calling it three times at different positions (you'll need parameters later, but for now just demonstrate the concept of reuse).

Direct Instruction: Mouse Events (20 minutes)

1. What is an Event?

Explain:

"An event is something that happens—like clicking a button, moving a mouse, or pressing a key. Event handlers are special functions that run automatically when an event happens."

Live Demo: onMousePress

Write on the projector:

def onMousePress(mouseX, mouseY):
    Circle(mouseX, mouseY, 20, fill='red')

Explain:

  • onMousePress is a special function CMU recognizes
  • It runs automatically every time you click
  • mouseX and mouseY are the click coordinates
  • You don't call this function—CMU calls it for you

2. onMouseMove

Demo a second example:

def onMouseMove(mouseX, mouseY):
    Circle(mouseX, mouseY, 10, fill='blue')

Explain: "This runs continuously as you move the mouse—it creates a trail effect."

3. Combining Functions and Events

Show how to call a custom function inside an event handler:

def drawStar(x, y):
    Star(x, y, 30, 5, fill='yellow')

def onMousePress(mouseX, mouseY):
    drawStar(mouseX, mouseY)

Key Point: "Now every click draws a star at that location."

Emphasize

  • Event handlers must have exact names (onMousePress, onMouseMove)
  • The parameters mouseX and mouseY are provided automatically
  • You can put any code inside an event handler

Guided Practice Setup (5 minutes)

Instructions for students:

  1. Log into CMU CS Academy
  2. Navigate to Unit 2.1 and complete the notes and exercises on Functions
  3. Then move to Unit 2.2 and complete the Mouse Events activities
  4. Work through in order—don't skip ahead

Remind students:

  • Read instructions carefully
  • Test code frequently
  • Ask for help if stuck for more than 5 minutes

Independent CMU Work Time (35 minutes)

Teacher Role:

  • Circulate and monitor progress
  • Provide one-on-one support
  • Identify common struggles
  • Encourage pair programming when appropriate

Common Questions to Ask Students:

  • "Why did you choose to make this a function?"
  • "What happens when you click here?"
  • "How does the program know where the mouse is?"

Student Tasks

In-Class Tasks

Students will:

  • Complete CMU CS Academy Unit 2.1 (Functions) notes and exercises
  • Complete CMU CS Academy Unit 2.2 (Mouse Events) notes and exercises
  • Create programs that:
    • Define and call at least one custom function
    • Respond to mouse clicks
    • Respond to mouse movement

Homework

Any unfinished Unit 2.1 or 2.2 CMU CS Academy lessons and exercises must be completed before the next class.

Checks for Understanding (Formative)

During work time, ask students:

  • "What does it mean to define a function?"
  • "How do you call a function after you've defined it?"
  • "What's the difference between onMousePress and onMouseMove?"
  • "Where do mouseX and mouseY come from?"
  • "Why might you want to put drawing code inside a function?"

Exit Ticket (last 5 minutes):

Students answer:

  • What is one reason programmers use functions?
  • What happens when you define an onMousePress function?

Common Misconceptions & Fixes

Misconception Fix
Thinking defining a function runs it automatically Demonstrate that def creates the function but you must call it to run it
Forgetting parentheses when calling a function Show error messages and emphasize functionName() syntax
Forgetting to indent function body Explain Python uses indentation to show what belongs to the function
Misspelling event handler names Stress that onMousePress must be spelled exactly right
Trying to call onMousePress manually Explain that event handlers are called automatically by the system
Not understanding where mouseX/mouseY come from Show that CMU provides these values automatically when events happen

Differentiation & Accommodations

For students who struggle

  • Provide function templates with clear structure
  • Allow extra time on simpler exercises before moving to events
  • Pair with a peer mentor
  • Focus on one concept at a time (functions OR events, not both immediately)

For advanced students

  • Challenge them to create functions with multiple shapes
  • Encourage combining both onMousePress and onMouseMove
  • Ask them to create an interactive drawing program
  • Invite them to help struggling peers

General supports

  • Visual function syntax reference
  • Event handler cheat sheet
  • Code structure templates
  • Frequent check-ins during work time

Assessment Notes

This lesson is primarily formative.

Evidence of mastery includes:

  • Successfully defining and calling custom functions
  • Creating working event handlers that respond to mouse actions
  • Completion of CMU CS Academy Unit 2.1 and 2.2 exercises
  • Verbal explanations during teacher check-ins

Formal assessment will come through upcoming Unit 2 projects and exercises.

Closure

"Today you learned two major programming concepts: functions and events. Functions let you organize and reuse code. Events let your programs respond to user actions."

"This is a huge step—your programs are now interactive. In the next lessons, we'll build on these skills to create even more complex and interesting programs."

Reminder: "Finish any incomplete Unit 2.1 and 2.2 work for homework."
