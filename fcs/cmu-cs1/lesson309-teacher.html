Teacher Lesson Plan

Lesson 309: 2.3 Properties – Guided Practice

Lesson Overview

Course: Fundamentals of Computer Science
Lesson Code: 309
Lesson Title: 2.3 Properties – Guided Practice
Class Length: 90 minutes
Curriculum Focus: CMU CS Academy CS1 – Unit 2.3 (Properties)

This lesson introduces and practices the concept of shape properties in CMU CS Academy. Students learn how to store shapes in variables and modify their properties such as position, size, color, and visibility after the shape has been created. Guided practice emphasizes reading and writing properties correctly and using them to create interactive behavior.

Objectives

Students will be able to:

  • Explain what a property is in the context of CMU CS Academy shapes.
  • Store shapes in variables in order to modify them later.
  • Change shape properties such as fill, border, width, height, centerX, and centerY.
  • Use properties inside event functions to create simple interactivity.

Materials & Setup

  • Computers with internet access
  • CMU CS Academy login credentials
  • Student note-taking materials
  • Teacher demo computer and projector
  • Access to CMU CS Academy Unit 2.3 activities

Teacher Preparation:

  • Open sample programs demonstrating property changes
  • Prepare a short live-coding example using a stored shape variable
  • Review which students may need additional support

Teacher Moves (Scripted Talking Points + Demo Steps)

Warm-Up (5 minutes)

Ask:

"Up to now, we have drawn shapes that never change. What if we want to move them, resize them, or change their color after they are created?"

Transition:

"Today you will learn how to control shapes after they are drawn using something called properties."

Direct Instruction (30 minutes)

Introduce the Idea of Properties

Talking points:

  • "Every shape has built-in characteristics called properties."
  • "Properties describe things like size, position, and color."
  • "If we store a shape in a variable, we can change those properties later."

Model Storing a Shape in a Variable

Demonstrate:

ball = Circle(200, 200, 40, fill='blue')

Explain:

  • The name ball now refers to this specific circle
  • We can change it after it is created

Changing Properties

Show examples one at a time:

ball.fill = 'red'
ball.radius = 60
ball.centerX = 300

Explain the pattern:

variableName.property = newValue

Emphasize:

  • Properties are changed using the dot (.) operator
  • Spelling and capitalization matter
  • Only valid properties can be changed

Using Properties with Events

Preview:

def onMousePress(mouseX, mouseY):
    ball.centerX = mouseX
    ball.centerY = mouseY

Explain how properties allow interactivity.

Guided Practice (10 minutes)

Work through a simple example together:

  1. Create a rectangle stored in a variable
  2. Change its color and size using properties
  3. Move it when the mouse is pressed

Have students follow along step by step.

Student Tasks

In-Class Tasks

Students will:

  • Log into CMU CS Academy
  • Open Unit 2.3 – Properties
  • Complete interactive notes and assigned exercises focused on:
    • Storing shapes in variables
    • Modifying properties
    • Using properties in event functions

Students should test changes frequently and use the Inspector tool to verify values.

Homework

Any unfinished Unit 2.3 exercises become homework so students are prepared for the next class.

Checks for Understanding (Formative)

During work time, ask:

  • "Why must we store a shape in a variable to change it later?"
  • "What does shape.centerX control?"
  • "How would you change the color of a shape after it is created?"
  • "What happens if you misspell a property name?"

Exit Questions:

  • Write the line of code to change a circle named sun to yellow.
  • What symbol is used to access a property?
  • Why are properties important for interactivity?

Common Misconceptions & Fixes

Misconception Fix
Trying to change a shape without storing it Emphasize need for variables
Using incorrect property names Review available properties
Forgetting quotes for string values Remind: colors need quotes
Believing properties can be changed before shape exists Reinforce order of execution
Confusing = with == Practice proper assignment syntax

Differentiation & Accommodations

For students who struggle

  • Provide starter code templates
  • Focus on a small set of properties first
  • Allow pair programming

For advanced students

  • Encourage combining multiple property changes
  • Add creative interactivity
  • Experiment with less common properties

General supports

  • Visual reference sheets
  • Step-by-step checklists
  • Extra modeling as needed

Assessment Notes

This lesson is formative and skill-building.

Evidence of mastery includes:

  • Correctly storing shapes in variables
  • Successfully modifying properties
  • Completing CMU exercises with correct results

No separate grade is assigned beyond normal CMU exercise completion.

Closure

"Properties are one of the most powerful ideas in this course. They allow programs to change and respond instead of staying static."

"Next we will continue using properties to build even more interactive programs."

Remind students to finish any remaining activities as homework.
