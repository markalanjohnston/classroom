Teacher Lesson Plan

Lesson 310: Unit 2 Exercises Catch-Up and Support Day

Lesson Overview

Course: Fundamentals of Computer Science
Lesson Code: 310
Lesson Title: Unit 2 Exercises Catch-Up and Support Day
Class Length: 90 minutes
Curriculum Focus: CMU CS Academy CS1 – Unit 2 (Functions, Mouse Events, Properties)

This lesson provides dedicated time for students to complete all Unit 2 exercises in CMU CS Academy. The focus is on consolidating understanding of functions, mouse events, and properties through practice and individualized support. The teacher provides brief review, troubleshooting assistance, and checks for understanding while students work independently on exercises. This catch-up day ensures all students complete Unit 2 before moving forward.

Objectives

Students will be able to:

  • Complete all assigned Unit 2 exercises in CMU CS Academy.
  • Apply knowledge of functions to organize code.
  • Use mouse events to create interactive programs.
  • Modify shape properties to change graphics after creation.
  • Debug common errors in Unit 2 concepts with increasing independence.

Materials & Setup

  • Computers with internet access
  • CMU CS Academy login credentials
  • Student note-taking materials
  • Teacher computer for quick demos
  • Whiteboard or projector for brief review

Teacher Preparation:

  • Review Unit 2 exercises to anticipate common issues
  • Check CMU CS Academy dashboard to identify students who are behind
  • Prepare quick review slides or notes for Unit 2 concepts
  • Have example solutions ready for common stuck points
  • Set expectations for work time behavior and help-seeking

Agenda (90 Minutes)

Time Activity
0 – 5 minAttendance and set expectations for the work day
5 – 15 minBrief Review of Unit 2 Key Concepts
15 – 80 minIndependent Work Time with Teacher Support
80 – 87 minQuick Checks for Understanding
87 – 90 minClosure and Homework Reminder

Teacher Moves (Scripted Talking Points + Demo Steps)

Opening and Expectations (5 minutes)

Set the Purpose:

"Today is a catch-up and work day. You will have most of the class to complete all Unit 2 exercises. This is your chance to finish strong and get help where you need it."

Explain the Structure:

  • "I will do a quick 10-minute review of Unit 2 concepts."
  • "Then you will work independently on CMU CS Academy for about an hour."
  • "I will be circulating to answer questions and help you debug."
  • "At the end, we'll do a quick check for understanding."

Behavioral Expectations:

"This is not free time. You should be actively working. If you finish early, help a classmate or work ahead. Stay focused and use this time well."

Brief Review of Unit 2 (10 minutes)

Quickly review the three main topics:

1. Functions (Unit 2.1)

Key idea: Functions organize code into reusable pieces.

Show quick example:

def drawStar():
    # code to draw a star

drawStar()  # Call the function to run it

Remind: Use def to define, then call by name.

2. Mouse Events (Unit 2.2)

Key idea: Programs can respond to mouse actions.

Show quick example:

def onMousePress(mouseX, mouseY):
    # code runs when mouse is clicked

Remind: These functions are automatically called by CMU.

3. Properties (Unit 2.3)

Key idea: Store shapes in variables and change them using properties.

Show quick example:

circle = Circle(200, 200, 50)
circle.fill = 'blue'
circle.centerX = 300

Remind: Must store in a variable first, then use dot notation.

Transition to Work Time:

"Those are the big ideas from Unit 2. Now you have time to practice and finish all the exercises. Raise your hand if you get stuck."

Independent Work Time (65 minutes)

Student Task:

Students work independently on CMU CS Academy Unit 2 exercises (2.1, 2.2, and 2.3).

Teacher Role During Work Time:

  1. Circulate actively – Monitor student progress, check screens, identify students who are stuck or off-task.
  2. Answer questions – Respond to raised hands, guide debugging, ask probing questions instead of giving answers directly.
  3. Offer targeted mini-lessons – If multiple students struggle with the same concept, pull a small group for a 3-minute reteach.
  4. Track completion – Use CMU dashboard to monitor which students are progressing and which need intervention.
  5. Encourage peer support – Students who finish early can help classmates (without doing the work for them).

Debugging Support Strategy:

When a student asks for help, use this approach:

  1. Ask: "What were you trying to do?"
  2. Ask: "What happened instead?"
  3. Ask: "What have you already tried?"
  4. Guide them to the error or issue rather than fixing it directly

Quick Checks for Understanding (7 minutes)

Before closure, ask a few quick review questions to the class:

  • "Why do we use functions?" (Organize code, reuse code)
  • "What does onMousePress do?" (Runs code when mouse is clicked)
  • "How do you change the color of a shape after it's created?" (Store in variable, use .fill property)

Call on individual students or use random selection. This helps reinforce key ideas.

Closure (3 minutes)

Wrap-Up Message:

"Unit 2 is a foundation for everything we do next. If you didn't finish all the exercises today, you must complete them as homework so you're ready for the next lesson."

Homework reminder: Any incomplete Unit 2 exercises must be finished before the next class.

Preview: "Next class, we'll move into a collaborative debugging challenge to strengthen your problem-solving skills."

Student Tasks

In-Class Tasks

Students will:

  • Log into CMU CS Academy
  • Navigate to Unit 2 and check completion status
  • Work through all incomplete or incorrect exercises in:
    • Unit 2.1 – Functions
    • Unit 2.2 – Mouse Events
    • Unit 2.3 – Properties
  • Test code frequently and debug errors
  • Seek help when stuck for more than 5 minutes
  • If finished early, help peers or begin Unit 3

Homework

Complete any remaining Unit 2 exercises not finished during class. All Unit 2 work must be done before the next lesson.

Checks for Understanding (Formative)

During work time, monitor for:

  • Can students define and call functions correctly?
  • Do students use event functions appropriately?
  • Can students modify shape properties after creation?
  • Are students testing and debugging their code?

Exit Questions (Oral or Written):

  • What is one thing you struggled with today, and how did you solve it?
  • Write the line of code to move a shape named box to x-coordinate 100.
  • Why must a shape be stored in a variable to change its properties later?

Common Misconceptions & Fixes

Misconception Fix
Students call a function before defining it Reinforce that definitions must come first
Students misspell event function names Check spelling and capitalization carefully
Students try to change properties without storing shape Emphasize need for variable assignment
Students confuse = (assignment) with == (comparison) Clarify difference with examples
Students copy code without understanding it Ask them to explain what each line does

Differentiation & Accommodations

For students who struggle

  • Provide extra scaffolding: starter code or step-by-step checklists
  • Break exercises into smaller sub-tasks
  • Allow pair programming with a stronger student
  • Offer extended time or reduce number of required exercises
  • Use visual aids and reference sheets

For advanced students

  • Encourage creative extensions to exercises
  • Assign peer tutoring or helper role
  • Allow them to begin Unit 3 if Unit 2 is complete
  • Challenge them to solve exercises in multiple ways

General supports

  • Circulate frequently to catch issues early
  • Provide immediate, specific feedback
  • Remind students to take breaks if frustrated
  • Normalize mistakes as part of learning to program

Assessment Notes

This lesson is primarily formative.

Evidence of progress includes:

  • Completion of Unit 2 exercises in CMU CS Academy
  • Correct use of functions, events, and properties
  • Ability to debug simple errors independently

Consider recording an engagement or completion grade based on student productivity during work time.

No formal assessment is given today, but monitor which students are falling behind and may need additional intervention.

Closure

"Today you had time to catch up and solidify your understanding of Unit 2. These concepts—functions, events, and properties—are the building blocks for everything we do next in this course."

"If you still have unfinished exercises, complete them tonight. Falling behind makes the next lessons much harder."

"Next class, we'll work together on a collaborative debugging challenge to strengthen your problem-solving skills."
