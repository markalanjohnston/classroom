Teacher Lesson Plan

Lesson 303: Unit 1.2 – Colors

Lesson Overview

Course: Fundamentals of Computer Science (Programming Semester)
Lesson Code: 303
Lesson Title: Unit 1.2 – Colors
Length: 90 minutes
Resource Base: CMU CS Academy CS1 Curriculum

This lesson introduces students to how color works in CMU CS Academy graphics programming. Students learn to apply built-in color names, adjust shape properties such as fill and opacity, and explore advanced RGB color values. The lesson builds on Unit 1 drawing skills and prepares students to create more expressive and visually accurate programs.

Objectives

By the end of the lesson students will be able to:

  • Use built-in color names to modify the appearance of shapes.
  • Apply the fill and opacity properties to CMU CS Academy graphics objects.
  • Explain how RGB color values work and use them correctly in programs.
  • Modify existing drawing programs to include custom colors.

Materials & Setup

  • Student computers with internet access
  • CMU CS Academy login credentials
  • Student note-taking materials
  • Projector or teacher computer for live demonstrations
  • Access to CMU CS Academy Unit 1.2 activities

Teacher Preparation:

  • Verify all students can log into CMU CS Academy
  • Open Unit 1.2 examples in advance
  • Prepare a sample starter program for live modeling

Teacher Moves (Scripted Talking Points + Demo Steps)

Warm-Up (5 min)

Teacher Prompt:

"Think about video games or apps you use every day. Why do colors matter? How would a program feel if everything were only black and white?"

Allow 2–3 student responses.

Transition:

"Today you are going to learn how programmers control color inside programs so you can make your drawings more realistic and creative."

Direct Instruction (30 minutes)

Key Concepts to Cover

  • Shapes have optional properties
  • The fill property controls interior color
  • Opacity controls transparency
  • Colors can be:
    • Named colors (like 'red', 'blue', 'green')
    • RGB values for custom colors

Live Demo Steps

On the projector:

Rect(50, 50, 100, 100, fill='blue')

Explain: "This rectangle is blue because we added the fill property."

Modify:

Rect(50, 50, 100, 100, fill='blue', opacity=50)

Explain opacity as a scale from 0 to 100.

Introduce RGB Values

Show:

Circle(200, 200, 50, fill=rgb(255, 0, 0))

Explain:

  • RGB stands for Red, Green, Blue
  • Values range from 0–255
  • Mixing these creates millions of colors

Model changing numbers and asking students to predict outcomes.

Think-Aloud Example

"Instead of hoping for the perfect named color, programmers can create exactly the color they want."

Student Tasks

In-Class Tasks

Students will:

  • Log into CMU CS Academy
  • Navigate to Unit 1.2 – Colors
  • Complete assigned exercises and examples involving:
    • Changing fill colors
    • Using opacity
    • Experimenting with RGB values

Students should attempt problems in order and use the Inspector tool to analyze solutions.

Homework

Any unfinished CMU CS Academy exercises from today become homework and must be completed before the next class meeting.

Checks for Understanding (Formative)

During work time:

Ask individual students:

  • "What does opacity 0 mean?"
  • "What color would rgb(0, 0, 255) create?"
  • "How do you change the border color instead of the fill?"

Exit Ticket (last 5 minutes):

Students answer on paper or digitally:

  • What property changes the inside color of a shape?
  • What range of values can RGB numbers have?
  • Write one line of code that draws a red circle.

Common Misconceptions & Fixes

Misconception Correction
Students think colors must be quoted like variables Explain strings must be in quotes: 'blue' not blue
RGB values can be any number Values must be between 0 and 255
Opacity 100 means transparent Opacity 100 means fully visible
Color names are case-sensitive Use lowercase names exactly as listed

Differentiation & Accommodations

Struggling learners

  • Provide a starter template program
  • Limit initial tasks to named colors only

Advanced learners

  • Encourage creating custom palettes with RGB
  • Experiment with gradients and complex designs

Accessibility

  • Allow color-blind friendly palettes
  • Provide written notes and examples

Assessment Notes

This lesson is primarily formative.

Evidence of mastery includes:

  • Correct use of fill and opacity
  • Successful use of RGB values
  • Completion of CMU exercises with passing autograder results

Closure

"Today you learned how to control color in your programs. Tomorrow you'll use these tools to create more complex and detailed drawings. Good programmers don't just make code work—they make it look good too."
