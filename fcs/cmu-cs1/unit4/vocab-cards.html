New Term Unit 4.1
Conditional Statement if  /  elif  /  else
Definition
A block of code that chooses which branch to execute based on a condition. Uses if, elif, and else to test different cases in order.
New Term Unit 4.1
elif else if
Definition
Short for "else if." Adds another condition that is only tested if all previous conditions were False. You can chain as many elifs as you need.
New Term Unit 4.1
else catch-all branch
Definition
The final branch in a conditional. Runs when none of the conditions above it were True. Does not take a condition — it always catches whatever is left.
New Term Unit 4.1
Boolean Condition True  /  False
Definition
A test that evaluates to either True or False. Written inside the parentheses of an if or elif to decide which branch of code runs.
New Term Unit 4.1
Event-Driven Programming respond to user actions
Definition
A style of programming where code runs in response to user actions — like mouse clicks or key presses — instead of running line by line from top to bottom.
Review Unit 4.1
Variable named storage
Definition
A named container that stores a value. The value can be read and changed throughout the program. Example: redLight = Circle(...)
Review Unit 4.1
def function definition
Definition
A named, reusable block of code created with the def keyword. It runs whenever it is called. Example: def onMousePress(x, y):
Review Unit 4.1
if conditional branch
Definition
Tests a condition. If the condition is True, the indented block of code runs. If False, it is skipped. Always the first branch in a conditional chain.
Review Unit 4.1
Property .fill  ·  .opacity  ·  .radius
Definition
An attribute that belongs to an object. Used to read or change an object's state. Example: redLight.opacity = 100 sets the opacity property of a circle.
Review Unit 4.1
onMousePress mouse event handler
Definition
An event handler that runs automatically every time the user clicks the mouse. Defined with def onMousePress(x, y): — the program calls it for you.
