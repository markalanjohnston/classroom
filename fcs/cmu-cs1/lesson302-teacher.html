Teacher Lesson Plan

Lesson 302: Course Introduction, Digital Citizenship, CMU Setup, Unit 1.1 Basics

Course Information

Course: Fundamentals of Computer Science – Programming Semester
Class Number: 302
Length: 90 Minutes
Curriculum: CMU CS Academy – CS1
Lesson Focus: Course Introduction, Digital Citizenship, CMU Setup, Unit 1.1 Basics

Lesson Overview

This lesson serves as the first true instructional day of the programming semester. The purpose is to:

  • Establish classroom procedures and expectations
  • Introduce digital citizenship and academic integrity
  • Ensure all students can access and use CMU CS Academy
  • Introduce the basic mechanics of programming
  • Begin Unit 1.1: Basics (drawing with code)

The lesson blends direct instruction with hands-on guided practice to set the tone for the semester-long instructional model.

Learning Objectives

Students will be able to:

  • Explain course expectations and classroom procedures.
  • Demonstrate understanding of digital citizenship and academic integrity in programming.
  • Successfully create and access a CMU CS Academy account and join the class.
  • Navigate the CMU CS Academy programming environment.
  • Write and run basic programs that draw simple shapes.

TEKS Alignment

This lesson directly supports the following Fundamentals of Computer Science TEKS:

  • (d)(1)(G): Legal and ethical responsibilities in computer science
  • (d)(3)(B): Debug and solve problems using effective strategies
  • (d)(4)(D): Demonstrate coding proficiency in a programming language
  • (d)(6)(B): Use system tools and technology resources appropriately

Materials and Technology

  • Student computers with internet access
  • Projector and teacher computer
  • CMU CS Academy class join code
  • Student login credentials
  • LMS access (Schoology/Canvas)

Vocabulary

  • IDE
  • Canvas
  • Autograder
  • Coordinates
  • Debugging
  • Syntax
  • Digital Citizenship

Instructional Procedures

1. Welcome and Course Framing (10 minutes)

Teacher Actions

  • Welcome students to the programming semester
  • Explain the purpose of the course
  • Review how this semester fits into the overall Fundamentals of Computer Science class
  • Introduce the daily class structure

Key Talking Points

  • Programming is learned through practice, not memorization
  • Mistakes and debugging are normal and expected
  • Class time will combine instruction and independent coding
  • Homework will be a regular part of the course

Student Outcomes

  • Understand expectations and course purpose
  • Recognize the structure of a typical class period

2. Digital Citizenship and Academic Integrity (10 minutes)

Mini-Lesson Topics

  • Ethical use of online resources
  • Collaboration vs. copying
  • Appropriate help-seeking
  • Plagiarism in coding
  • Respectful online behavior

Discussion Prompts

  • "What is the difference between helping someone and giving them your code?"
  • "Is it okay to search for syntax help online?"
  • "Why is copying code harmful to learning?"

Key Expectations to Establish

  • Students may discuss ideas but must write their own code
  • Copy/paste from classmates or websites is not acceptable
  • Asking for help is encouraged
  • Integrity is essential in computer science

3. CMU CS Academy Account Setup (15 minutes)

Step-by-Step Process

  • Direct students to the CMU CS Academy website
  • Walk students through account creation
  • Provide class join code
  • Verify enrollment on teacher dashboard

Teacher Responsibilities

  • Circulate and assist
  • Troubleshoot login issues
  • Confirm every student can access Unit 1

Success Criteria

  • 100% of students logged in
  • All students able to open Unit 1 lessons

4. IDE Tour and Demonstration (15 minutes)

Demonstrate and Explain

  • Code editor window
  • Run button
  • Graphics canvas
  • Autograder feedback
  • Inspector tool

Emphasize

  • Programs are written line by line
  • Output appears visually on the canvas
  • Errors are part of the process
  • The Inspector helps analyze shapes and coordinates

Modeling

Teacher writes a simple program live and shows:

  • How changing numbers affects output
  • How to fix simple syntax mistakes
  • How to rerun and test code

5. Unit 1.1 Basics Instruction (20 minutes)

Core Concepts Introduced

  • Coordinates on the canvas
  • Drawing shapes with code
  • Basic shape commands:
    • Rect(left, top, width, height)
    • Circle(centerX, centerY, radius)
    • Oval(centerX, centerY, width, height)
    • Line(x1, y1, x2, y2)

Guided Demonstration

Teacher models:

Rect(50, 50, 100, 150)
Circle(200, 200, 40)

Discuss:

  • What each number represents
  • How to modify shapes
  • Trial-and-error problem solving

6. Guided Independent Practice (15 minutes)

Student Tasks

  • Begin Unit 1.1 Notes
  • Complete Unit 1.1 Exercises
  • Practice running and modifying code

Teacher Role

  • Circulate actively
  • Provide one-on-one support
  • Encourage use of Inspector tool
  • Reinforce debugging mindset

7. Closure and Formative Check (5 minutes)

Exit Ticket Questions

Students respond:

  • What is one shape you learned to draw today?
  • What does the Run button do?
  • What should you do if your code doesn't work?

Teacher Uses Responses To

  • Gauge student understanding
  • Identify students needing extra support
  • Adjust pacing for next lesson

Assessment Strategy

Formative Assessment

  • Observation of student participation
  • Completion of CMU setup
  • Progress in Unit 1.1 exercises
  • Exit ticket responses

No formal grade is assigned during this lesson.

Differentiation

  • Advanced students: encouraged to explore additional exercises
  • Struggling students: receive step-by-step scaffolding
  • Peer support: pair programming for initial help
  • Visual learners: heavy use of live demonstrations

Classroom Management Considerations

  • Establish norms for help-seeking
  • Normalize debugging and mistakes
  • Monitor off-task computer behavior
  • Reinforce procedures for saving work

Homework

  • Complete any remaining Unit 1.1 exercises
  • Come prepared to begin Unit 1.2 – Colors next class

Accommodations and Supports

  • Extended time as needed
  • Read-aloud support for instructions
  • One-on-one assistance during work time
  • Alternative seating or grouping as required

Teacher Reflection Questions

After class, the teacher should consider:

  • Were all students able to log in successfully?
  • How many completed Unit 1.1 exercises?
  • Which concepts caused the most confusion?
  • Does pacing need adjustment for Class 303?
