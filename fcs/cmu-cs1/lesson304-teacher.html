Teacher Lesson Plan

Lesson 304: Unit 1.3 – Shapes

Lesson Overview

Course: Fundamentals of Computer Science
Lesson Code: 304
Lesson Title: Unit 1.3 – Shapes
Class Length: 90 minutes
Curriculum Source: CMU CS Academy CS1 – Unit 1

This lesson expands student understanding of the core drawing tools available in CMU CS Academy. Students learn additional shape commands, the parameters each shape requires, and how coordinates, width, height, and radius affect the final drawing. The focus is on recognizing when to use different shapes and accurately placing them on the canvas.

Objectives

Students will be able to:

  • Identify and use the primary shape-drawing commands in CMU CS Academy.
  • Explain the difference between shapes defined by radius and shapes defined by width and height.
  • Correctly position shapes on the canvas using x and y coordinates.
  • Create simple drawings that combine multiple types of shapes.

Materials & Setup

  • Computers with internet access
  • CMU CS Academy student login credentials
  • Student note-taking materials
  • Projector or teacher computer for demonstrations
  • Access to Unit 1.3 activities in CMU CS Academy

Teacher Preparation:

  • Confirm all students can log in to CMU CS Academy
  • Open example programs from Unit 1.3 in advance
  • Prepare a sample drawing program for live modeling

Teacher Moves (Scripted Talking Points + Demo Steps)

Warm-Up (5 minutes)

Ask students:

"Look around the room. How many everyday objects can you describe using only simple shapes like rectangles, circles, and lines?"

Allow a few responses.

Transition:

"Computer graphics are built the same way. Complex pictures come from combining simple shapes. Today you will learn the main shapes available in CMU CS Academy and how to control them."

Direct Instruction (35 minutes)

1. Introduce Available Shapes

Explain that Unit 1 uses several basic drawing commands, including:

  • Circle
  • Rect
  • Oval
  • Line
  • Polygon
  • Star
  • Label

Clarify:

"Every picture you create in CMU CS Academy is built from these basic pieces."

2. Shapes Defined by Radius

Demonstrate:

Circle(200, 200, 50)
Star(100, 100, 40, 5)

Explain:

  • These shapes use a center point and a radius
  • The radius controls size in all directions

Talking Point:

"Radius-based shapes grow outward equally from their center."

3. Shapes Defined by Width and Height

Demonstrate:

Rect(50, 50, 120, 80)
Oval(200, 200, 100, 60)

Explain:

  • These shapes start from a corner or center and use width and height
  • Rectangles are drawn from the top-left corner by default

4. Coordinate System Review

Use the canvas to remind students:

  • (0,0) is the top-left corner
  • x increases to the right
  • y increases downward

Ask guiding questions:

"If I want a shape lower on the screen, which value should I change?"

5. Combining Shapes

Model building a simple picture using multiple commands:

Rect(50, 150, 300, 100)
Circle(200, 150, 40)
Line(50, 150, 350, 150)

Emphasize:

"Programs run from top to bottom, so order matters."

Student Tasks

In-Class Work

Students will:

  • Log into CMU CS Academy
  • Navigate to Unit 1.3 – Shapes
  • Work through interactive notes and assigned exercises
  • Use multiple shape types to complete the practice tasks

Focus areas:

  • Using correct parameters
  • Positioning shapes accurately
  • Recognizing when to use each type of shape

Homework

Any unfinished Unit 1.3 CMU CS Academy lessons or exercises must be completed before the next class session.

Checks for Understanding (Formative)

During work time, ask:

  • "What values does the Rect command need?"
  • "How is a Circle different from an Oval?"
  • "Which part of the screen is coordinate (0,0)?"
  • "How do you control the size of a Star?"

Quick Exit Questions:

  • Which shape uses radius instead of width and height?
  • What do the first two numbers in a shape command represent?
  • Why does order matter when drawing shapes?

Common Misconceptions & Fixes

Misconception Fix
Students think all shapes start at center Explain that Rect starts at top-left by default
Confusing width/height with x/y Reinforce that x and y are position, not size
Belief that order of code doesn't matter Demonstrate layering through examples
Mixing up radius with diameter Model how radius controls size directly
Using wrong number of parameters Review required inputs for each shape

Differentiation & Accommodations

For students who struggle

  • Provide partially completed starter code
  • Allow them to focus on only Circle and Rect first
  • Offer coordinate grid templates

For advanced students

  • Challenge them to recreate a complex picture
  • Encourage use of Polygon and Star creatively
  • Ask them to design a custom scene

General supports

  • Pair programming
  • Visual coordinate references
  • Step-by-step checklists

Assessment Notes

Assessment in this lesson is formative.

Evidence of understanding includes:

  • Correct use of multiple shape commands
  • Accurate placement on canvas
  • Successful completion of CMU exercises

Formal grading will come later through creative tasks and unit assignments.

Closure

"Today you learned the building blocks of computer graphics. Every complex image in this course starts with these simple shapes. As you continue, you'll combine them in more creative and powerful ways."

Remind students: "Finish any remaining exercises for homework so you are ready for the next lesson."
