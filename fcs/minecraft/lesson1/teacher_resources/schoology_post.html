Minecraft MakeCode Lesson 1

Sequencing, Events, and Agent Movement

Today’s Big Idea

Today we will use Minecraft MakeCode to control the Agent with code. You will write instructions that run in order, test your program, find mistakes, and improve your code.

This lesson connects directly to programming concepts you will see next year in AP Computer Science Principles.

Learning Objectives

By the end of class, you should be able to:

  • Explain why the order of code matters.
  • Use an event, such as a chat command, to start a program.
  • Write a sequence of commands that moves the Minecraft Agent.
  • Test and debug code when the Agent does not do what you expected.
  • Modify starter code to create your own Minecraft build.

Key Vocabulary

Term Meaning
Sequence The order that instructions run in a program.
Event Something that starts code, such as typing a command.
Agent A Minecraft character that follows your code instructions.
Debugging Finding and fixing mistakes in code.

What We Will Do Today

  1. Review how computers follow instructions exactly as written.
  2. Open Minecraft MakeCode and connect to the Agent.
  3. Copy and run starter code.
  4. Modify the code to change what the Agent does.
  5. Create your own maze entrance, path, or simple structure.
  6. Submit evidence of your work.

Starter Code

We will begin with code similar to this. Do not worry if it does not make sense yet. We will break it down together. 

def on_chat():
    agent.set_item(STONE, 1, 1)
    agent.move(FORWARD, 5)
    agent.turn(LEFT_TURN)
    agent.move(FORWARD, 5)
    agent.place(DOWN)
player.on_chat("road", on_chat)

After we test it, your job will be to change the code and make the Agent do something new.

Success Criteria

Your work is successful when you can show:

  • Your Agent moves using code.
  • Your program starts from a chat command.
  • Your code includes several steps in the correct order.
  • You changed the starter code instead of only copying it.
  • You can explain at least one bug you fixed.

What You Will Submit

  • A screenshot of your code.
  • A screenshot of what your Agent built or changed in Minecraft.
  • A short reflection answering:
    What did your code do, and what did you have to fix?

Essential Question

How does the order of instructions affect what a program does?
