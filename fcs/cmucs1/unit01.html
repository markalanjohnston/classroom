Unit 1: Creating Drawings

Unit Overview

In this unit, you'll learn the fundamentals of creating visual graphics using shapes, colors, and opacity. You'll explore various shape functions, understand optional parameters, and master the use of the Inspector tool to recreate precise drawings.

Shapes You Can Draw

Shapes with Radius

Circle

Circle(centerX, centerY, radius)

Star

Star(centerX, centerY, radius, points)

Regular Polygon

RegularPolygon(centerX, centerY, radius, points)

Draws a shape with the specified number of points evenly spaced around the circle.

Shapes with Width/Height

Rectangle

Rect(left, top, width, height)

Rectangles are drawn from their left-top coordinate by default.

Oval

Oval(centerX, centerY, width, height)

Miscellaneous Shapes

Line

Line(x1, y1, x2, y2)

Label

Label(value, centerX, centerY)

'value' is the text displayed by the label.

Polygon

Polygon(x1, y1, x2, y2, x3, y3, ...)

Like connect-the-dots. Each (x, y) pair is a dot. Lines are drawn between consecutive points.

Polygon Note

There is no upper limit to the number of points in a polygon, but it will not be drawn unless there are at least three points. Having 0, 1, or 2 points will not raise an error.

Optional Parameters

In addition to required parameters that determine position (x, y) and size (radius / width, height), shapes have optional parameters that you can use to modify their appearance.

Example: Rectangle with All Optional Parameters

Rect(left, top, width, height, fill='black', border=None, borderWidth=2, opacity=100, rotateAngle=0, dashes=False, align='left-top', visible=True)

Advanced Color Options

Gradients

Rather than a solid color, fill can also be set equal to a gradient—one color transitioning gradually to one or more other colors. You can change the orientation of the gradient.

RGB Values

While we have many named colors, computers can express many more (16 million actually). To use these other colors, you'll need to use RGB values.

# RGB value example Circle(200, 200, 50, fill=rgb(150, 75, 200))

Understanding RGB

An RGB value is composed of three numbers (all between 0 and 255):

  • First value = intensity of Red
  • Second value = intensity of Green
  • Third value = intensity of Blue

Reference colors:

  • rgb(255, 0, 0) = Pure red
  • rgb(0, 255, 0) = Pure green
  • rgb(0, 0, 255) = Pure blue
  • rgb(0, 0, 0) = Black
  • rgb(255, 255, 255) = White

The Inspector

Once something has been drawn on the canvas, hover your mouse over shapes to see highlighted points and a grey box in the top-right corner with information. This is called the Inspector and is vital to CMU CS Academy.

Important

Once we get further into the course and make interactive canvases, hold Control to activate the inspector.

Using the Inspector

Students can use the inspector on the solution view of the canvas to gather information about what to code. The limited information provided is meant to promote problem-solving skills rather than feeding answers.

Important Tips

Inspector Tips

  • All properties are listed from back to front - The first radius value corresponds to the shape furthest back
  • Pick isolated points when possible - Points belonging to only one shape avoid confusion
  • The inspector is your best friend - Use it extensively to match drawings exactly

The importance of the inspector cannot be overstated. Without it, many exercises would be nearly impossible to recreate!

Common Optional Parameters

  • fill - The color of the inside of a shape
  • opacity - How opaque a shape is (0-100 scale)
  • border - The color of a shape's border
  • borderWidth - The width of a shape's border, if it has one
  • dashes - Whether the border is dashed. False for no dashes, otherwise True or (dashWidth, gapWidth)

Troubleshooting & Advice

Exercise Drawing Looks Correct

Drawings must be exact matches. Note that any solution that accomplishes this will pass the autograder. For instance, an identical rectangle may be drawn with both the Rect function and the Polygon function!

Common Problems

  • Gradient directions are incorrect
  • Incorrect number of shapes (Are you drawing extra hidden shapes? Redundant shapes?)
  • Points don't match the solution
  • Incorrect colors (spelling and/or RGB values)
  • Incorrect property values (radius, border, etc.)

Nothing is Happening

Check the console (area underneath the canvas) for an error message. If there is an error, look at that line number, as well as before and after, to see if there are any syntax issues. Make sure your code is not in comments.

Not Sure Where to Start

Try drawing the image on paper, and translate your first steps into code with the inspector.

Not Sure How to Get It to Look Right

Layering is often used to cover up parts of shapes that aren't wanted. Code runs from top to bottom, so later lines of code get drawn on top of earlier ones.
