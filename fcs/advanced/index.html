Beyond CMU CS Academy

Run your own CMU programs locally with VS Code — no internet, no cmu account needed.

Learning Goals

EQ: How do real software developers run Python programs without a website doing it for them?

I will...

  • Open VS Code, create a Python file, and run it from the editor.
  • Install the cmu-graphics library on this lab machine.
  • Take a program I already wrote in CMU CS Academy and adapt it to run locally with two small changes.
  • Run my own CMU CS Academy program offline, in a real editor.

Why This Lesson

So far, every program you've written this year has lived inside a website — CMU CS Academy in your browser. That is great for learning, but it is not how real software is built.

Real developers write code in a code editor on their own machine, save it as a file, and run it from a terminal. The good news: your CMU CS Academy code already works locally with two tiny changes. By the end of this lesson, you'll have one of your own programs running on your computer with no internet connection at all.

The big idea: CMU CS Academy uses an open-source library called cmu-graphics. The same library is installable on any computer with Python. That means your CMU programs can leave the browser.

PART 1 Open VS Code & Make a Project Folder

VS Code (Visual Studio Code) is a free, professional code editor made by Microsoft. It is already installed on the lab machines.

  1. Open VS Code from the Start menu. (If it is not there, open Software Center and install it.)
  2. Go to File → Open Folder... and create a new folder called fcs-advanced in your Documents. Open that folder.
  3. If VS Code asks "Do you trust the authors of the files in this folder?" — click Yes, I trust the authors.
  4. Click the Extensions icon on the left sidebar (it looks like four squares: ). Search for Python — install the one published by Microsoft.
  5. Open the Command Palette with Ctrl + Shift + P, type Python: Select Interpreter, and press Enter. Pick the Python that shows up. (If nothing shows up, ask your teacher — Python may not be installed yet.)

Check your work

You have a folder open in VS Code, the Python extension is installed, and an interpreter is selected (visible at the bottom-right of the VS Code window).

PART 2 Install cmu-graphics (One Time)

You only need to do this once per machine. The library is what lets Rect, Circle, Polygon, etc. work outside the browser.

  1. In VS Code, open a terminal: Terminal → New Terminal (or press Ctrl + `).
  2. Type the following command and press Enter:
    python -m pip install cmu-graphics
  3. If you see a permission error, try this version instead:
    python -m pip install --user cmu-graphics
  4. Confirm the install with:
    python -m pip show cmu-graphics
    You should see a block of text starting with Name: cmu-graphics.
Stuck on install? If both commands fail, you may be on a managed network with restrictions. Stop and ask your teacher — this is the same kind of finding the deployment lab in /pit/python-deployment-lab is designed to surface.

PART 3 The Two-Line Conversion

Here is the entire trick to running a CMU CS Academy program locally:

Add ONE line at the top:  from cmu_graphics import *
Add ONE line at the bottom:  cmu_graphics.run()

That is it. The body of your program — every Rect, Circle, def onKeyPress, etc. — stays exactly the same.

Why it works

  • The CMU CS Academy website automatically does the import for you behind the scenes — on your own computer, you have to do it yourself.
  • The website also automatically opens a window when you press Run — on your own computer, the cmu_graphics.run() line is what opens the window.

PART 4 Worked Example: Traffic Light Game

Let's verify everything works with a known-good program before you try your own.

  1. In VS Code, click the New File icon next to your folder name in the sidebar (or File → New File) and name it traffic_light.py.
  2. Copy the entire program below into that file: 
    from cmu_graphics import *

app.background = gradient('deepSkyBlue', 'lightSkyBlue', start='top')

Rect(0, 150, 400, 250, fill=gradient('forestGreen', 'limeGreen', start='bottom'))

# traffic light
Line(200, 75, 200, 65)
Line(0, 50, 200, 65)
Line(400, 50, 200, 65)
Rect(175, 75, 50, 100, fill='orange')
redLight = Circle(200, 100, 20, fill='red')
greenLight = Circle(200, 150, 20, fill='gray')
msg = Label('Stop!', 200, 350, size=20, bold=True)

# road
Rect(0, 200, 400, 100)
Line(0, 250, 400, 250, fill='yellow', lineWidth=3, dashes=True)

# car
car = Polygon(0, 211, 3, 210, 12, 210, 15, 211, 30, 211, 33, 210, 45, 210, 48, 220,
              48, 230, 45, 240, 33, 240, 30, 239, 15, 239, 12, 240, 3, 240, 0, 239,
              fill=gradient('royalBlue', 'mediumBlue'))
frontWindow = Polygon(40, 215, 42, 225, 40, 235, 30, 234, 30, 216)
backWindow = Rect(5, 215, 5, 20)
top = Polygon(10, 215, 30, 216, 30, 234, 10, 235, fill=None, border='black',
              borderWidth=1)

def moveCar(leftPosition):
    car.left = leftPosition
    frontWindow.left = leftPosition + 30
    backWindow.left = leftPosition + 5
    top.left = backWindow.right

def changeLight(redFill, greenFill, msgValue):
    redLight.fill = redFill
    greenLight.fill = greenFill
    msg.value = msgValue

def onKeyPress(key):
    # Changes the color of the light.
    if (key == 'enter'):
        if (msg.value == 'Go!'):
            changeLight('red', 'gray', 'Stop!')
        else:
            changeLight('gray', 'green', 'Go!')

def onKeyHold(keys):
    # Move the car forward or back to the starting point according to the
    # light color while the game is playing.
    if (msg.value != 'Car wins!'):
        if ('space' in keys):
            if (redLight.fill == 'red'):
                moveCar(0)
            else:
                moveCar(car.left + 3)
                if (car.right > 400):
                    msg.value = 'Car wins!'
                    msg.fill = car.fill

cmu_graphics.run()
  3. Save the file: Ctrl + S.
  4. Run it. Three ways to do this — pick whichever is easiest:
    • Click the ▶ Run arrow at the top-right of the editor window.
    • Press F5 and choose Python File.
    • In the terminal, type python traffic_light.py and press Enter.
  5. A graphics window should open showing a traffic light, a road, and a blue car. Press Enter to toggle the light, hold Space to drive when it's green, and try to win!

Check your work

The graphics window opened, the traffic light is visible, and the car moves with Space when the light is green. If yes — your environment works. Move on.

PART 5 Now Do It With Your Own CMU Program

Pick one of the programs you wrote earlier this year in CMU CS Academy. (A drawing, a small game, an animation — whatever you have.)

  1. Log into CMU CS Academy in your browser and open the program you want to bring over.
  2. Select all of the code in the CMU editor (Ctrl + A) and copy it (Ctrl + C).
  3. In VS Code, create a new file in your folder. Give it a sensible name with the .py extension — for example my_drawing.py. Don't use spaces or hyphens; use underscores.
  4. Paste your code into the new file (Ctrl + V).
  5. At the very top of the file, add this line:
    from cmu_graphics import *
  6. At the very bottom of the file, add this line:
    cmu_graphics.run()
  7. Save the file (Ctrl + S) and run it the same way as Part 4.
Most programs work with no other changes. The same shapes, colors, event handlers, and variable names you used in CMU CS Academy work locally. If something behaves differently, write down exactly what's different — that's part of the deliverable.

Troubleshooting

"ModuleNotFoundError: No module named 'cmu_graphics'" The library isn't installed for the Python you're running. Re-run python -m pip install cmu-graphics in the VS Code terminal. Make sure the interpreter selected at the bottom-right of VS Code is the same Python you ran pip with.
A graphics window flashes and closes immediately You forgot the cmu_graphics.run() line at the bottom. Without it, the program ends before the window has anything to show.
"SyntaxError" on the very first line that uses Rect / Circle / etc. You forgot from cmu_graphics import * at the top. Python doesn't know what those names mean until you import them.
My program uses onMousePress / onStep / etc. — do those still work? Yes. All of CMU CS Academy's named event handlers (onMousePress, onMouseMove, onKeyPress, onKeyHold, onStep) are detected by cmu-graphics and called automatically — just like in the browser.
"pip is not recognized" or "python is not recognized" Python isn't on your PATH. Open a brand-new terminal (the PATH only updates for new shells) and try again. If it still fails, see the deployment lab in /pit/python-deployment-lab for fallback install paths.
