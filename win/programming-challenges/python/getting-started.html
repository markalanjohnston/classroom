What is a Programming Language?

A programming language is a set of rules in which symbols represent actions. Programming languages follow step-by-step instructions that a computer executes in order for a project to run.

In this course, VEXcode VR Python is the programming language for a VR Robot. Just like English has grammar rules for writing sentences, Python has syntax rules for writing code.

Think About It: Programming languages are like recipes. Each instruction must be written clearly and in the correct order for the computer to understand what to do.

The VEXcode VR interface with a Python project open showing the toolbox and playground

Key Characteristics of Programming Languages

  • Precise: Every character, space, and punctuation mark matters
  • Logical: Commands execute in a specific order from top to bottom
  • Repeatable: The same code will always produce the same result
  • Modifiable: You can change commands to change the robot's behavior

What is a Behavior?

Behaviors are the actions performed or to be performed by a robot. Moving forward, stopping, turning, looking for an obstacle - these are all behaviors. Behaviors are defined by the programming language.

To change the behaviors of a VR Robot, you can change the commands in the project. Each command you write tells the robot to perform a specific behavior.

Examples of Robot Behaviors:
  • Moving forward or backward
  • Turning left or right
  • Stopping at a specific location
  • Detecting obstacles with sensors
  • Picking up or releasing objects

How Commands Control Behavior

Every line of code you write changes what the robot does. Let's look at a simple example:

# This command makes the robot drive forward drivetrain.drive_for(FORWARD, 200, MM)

The drivetrain.drive_for command in the toolbox and coding interface

In this command, we're telling the drivetrain (the robot's wheels) to drive in the FORWARD direction for 200 millimeters. If we change the number to 400, the robot will drive twice as far. If we change FORWARD to REVERSE, the robot will drive backwards instead!

Understanding Python Syntax

Now that you understand what programming languages and behaviors are, let's learn the specific rules for writing Python code. Python has some unique features that make it different from other languages.

1. Indentation Matters

Unlike many other programming languages that use brackets {}, Python uses indentation (spaces at the beginning of lines) to show which code belongs together.

Blocks

In block coding, blocks fit together like puzzle pieces to show relationships:

when started
  repeat 4
    drive forward
    turn right

Python

In Python, indentation shows which code is inside the loop:

for i in range(4): drivetrain.drive_for(FORWARD, 200, MM) drivetrain.turn_for(RIGHT, 90, DEGREES)
Common Mistake: Forgetting indentation or using inconsistent spacing will cause errors! VEXcode VR automatically indents for you, but it's important to understand why indentation matters.

2. Using Methods and Functions

In Python, we use methods to tell objects (like the drivetrain) what to do. A method is called using a dot (.) between the object and the action:

# Format: object.method(parameters) drivetrain.drive_for(FORWARD, 200, MM) # ↑ ↑ ↑ ↑ ↑ # object dot method parameters

3. Parameters and Arguments

Parameters are the values inside parentheses that give the method more information. They tell the robot exactly how to perform the behavior.

# Three parameters: direction, distance, and units drivetrain.drive_for(FORWARD, 200, MM) # Three parameters: direction, angle, and units drivetrain.turn_for(RIGHT, 90, DEGREES)

4. Comments

Comments are notes you write in your code that Python ignores. They help you and others understand what your code does. In Python, comments start with a # symbol:

# This is a comment - it won't run drivetrain.drive_for(FORWARD, 200, MM) # This is also a comment
Best Practice: Use comments to explain WHY you're doing something, not just WHAT you're doing. This helps when you (or someone else) reads your code later!

5. Case Sensitivity

Python is case sensitive, which means uppercase and lowercase letters are treated differently. FORWARD is not the same as forward or Forward.

# CORRECT - uses the proper constant drivetrain.drive_for(FORWARD, 200, MM) # WRONG - lowercase 'forward' will cause an error drivetrain.drive_for(forward, 200, MM) # ❌ Error!

Practice Examples

Now that you understand the basics of Python syntax, let's look at some practical examples you can try in VEXcode VR.

Example 1: Drive Forward and Stop

The simplest program - just drive forward for a short distance:

# Drive forward 300 millimeters drivetrain.drive_for(FORWARD, 300, MM)

What it does: The robot drives forward 300mm (about 12 inches) and stops.

Example 2: Drive and Turn

Combine two commands to make the robot drive and then turn:

# Drive forward drivetrain.drive_for(FORWARD, 200, MM) # Turn right 90 degrees drivetrain.turn_for(RIGHT, 90, DEGREES)

What it does: The robot drives forward 200mm, stops, then turns right 90 degrees.

Example 3: Draw a Square

Use a loop to repeat commands and draw a square shape:

# Draw a square with 200mm sides for i in range(4): drivetrain.drive_for(FORWARD, 200, MM) drivetrain.turn_for(RIGHT, 90, DEGREES)

What it does: The robot repeats the drive-and-turn sequence 4 times, creating a square path. Notice how the indented lines run 4 times!
