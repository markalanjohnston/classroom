Basic Movement - VEX VR Python

Learn to control your VR Robot's movement! In this lesson, you'll master the fundamental drivetrain commands to move forward, backward, and turn your robot with precision.

What You'll Build: By the end of this lesson, you'll complete the Castle Crasher Challenge - programming your robot to navigate around a playground and knock over all the castle buildings using movement commands!

Learning Outcomes

By the end of this lesson, you will be able to:

  • Use the drive_for() command to drive the VR Robot a specific distance
  • Change parameters in the drive_for() command to move forward and reverse
  • Use the set_drive_velocity() command to control the speed of the drivetrain
  • Use the turn_for() command to turn the VR Robot right or left
  • Use the turn_to_heading() command to turn the VR Robot to a specific heading
  • Apply drivetrain commands to solve complex navigation challenges
  • Understand the importance of command sequencing in robot programming

Driving Forward and Reverse

The most basic movement command is drive_for(), which tells the VR Robot to drive in a specific direction for a specific distance. This command is your robot's primary way to move around the playground.

The drive_for() Command

# Basic syntax for drive_for command drivetrain.drive_for(FORWARD, 200, MM)

This command has three parts (parameters):

Parameter Description Options
Direction Which way the robot should move FORWARD or REVERSE
Distance How far the robot should travel Any number (integer or decimal)
Units The measurement unit for distance MM (millimeters) or INCHES

Blocks

In block coding, you select options from dropdown menus:

drive for [FORWARD] [200] [mm]

Python

In Python, you type the parameters separated by commas:

drivetrain.drive_for(FORWARD, 200, MM)

Moving Forward

To make your robot drive forward, use FORWARD as the first parameter:

# Drive forward 800 millimeters drivetrain.drive_for(FORWARD, 800, MM)

Moving in Reverse

To make your robot drive backward, change the direction to REVERSE:

# Drive backward 800 millimeters drivetrain.drive_for(REVERSE, 800, MM)
Pro Tip: The distance value must be the same for forward and reverse if you want your robot to return to its starting position. For example, if you drive forward 800mm, you need to reverse 800mm to get back to where you started!

Combining Forward and Reverse

You can chain multiple commands together to create more complex movements:

# Drive forward, then return to start drivetrain.drive_for(FORWARD, 800, MM) drivetrain.drive_for(REVERSE, 800, MM)

Understanding Parameters

Parameters are the values you pass to a command to tell it exactly what to do. Think of them as the "settings" for your command. Let's explore how changing parameters affects robot behavior.

Distance Values

The distance parameter can accept both whole numbers (integers) and decimals (floats):

# Using whole numbers drivetrain.drive_for(FORWARD, 200, MM) # Using decimals for more precision drivetrain.drive_for(FORWARD, 150.5, MM)

Unit Options

You can measure distance in millimeters (MM) or inches (INCHES):

# Using millimeters drivetrain.drive_for(FORWARD, 200, MM) # Using inches - approximately the same distance drivetrain.drive_for(FORWARD, 8, INCHES)
Quick Reference: 1 inch = 25.4 millimeters. Most VEX examples use millimeters (MM) because they provide more precision for small movements.

Controlling Speed

By default, the robot moves at 50% speed. You can change this using the set_drive_velocity() command:

# Set the drive velocity to 100% (fastest) drivetrain.set_drive_velocity(100, PERCENT) # Now all drive commands will use this speed drivetrain.drive_for(FORWARD, 800, MM)
Important: The set_drive_velocity() command accepts values from 0 to 100. Once set, it applies to all subsequent drivetrain commands until you change it again.

Example: Speed Comparison

# Slow movement (25% speed) drivetrain.set_drive_velocity(25, PERCENT) drivetrain.drive_for(FORWARD, 500, MM) # Fast movement (100% speed) drivetrain.set_drive_velocity(100, PERCENT) drivetrain.drive_for(FORWARD, 500, MM)

What happens: The robot drives the same distance twice, but the second movement is much faster!

Turning the Robot

Now that you can move your robot forward and backward, let's learn how to turn! VEX VR Python provides two commands for turning: turn_for() and turn_to_heading().

The turn_for() Command

The turn_for() command rotates the robot a specific number of degrees in a given direction:

# Turn right 90 degrees drivetrain.turn_for(RIGHT, 90, DEGREES) # Turn left 45 degrees drivetrain.turn_for(LEFT, 45, DEGREES)
Parameter Description Options
Direction Which way the robot should turn RIGHT or LEFT
Angle How many degrees to turn Any number (0-360 and beyond)
Units The measurement unit for angle DEGREES

The turn_to_heading() Command

The turn_to_heading() command turns the robot to face a specific compass direction (heading). This is useful when you need the robot to face an exact direction, regardless of where it's currently facing.

Understanding Headings: Think of headings like a compass:
  • 0 degrees = North (straight ahead from start)
  • 90 degrees = East (robot's right)
  • 180 degrees = South (behind)
  • 270 degrees = West (robot's left)
# Turn to face 90 degrees (east) drivetrain.turn_to_heading(90, DEGREES) # Turn to face 270 degrees (west) drivetrain.turn_to_heading(270, DEGREES) # Turn back to facing forward (north) drivetrain.turn_to_heading(0, DEGREES)
When to Use Which: Use turn_for() when you want to turn relative to the robot's current position. Use turn_to_heading() when you need to face an absolute direction, especially useful after collisions when the robot might be knocked off course!

Controlling Turn Speed

Just like driving, you can control how fast the robot turns using set_turn_velocity():

# Set turn velocity to 100% (fastest) drivetrain.set_turn_velocity(100, PERCENT) # This turn will be fast drivetrain.turn_for(RIGHT, 90, DEGREES)

Blocks

Turning with blocks:

turn for [RIGHT] [90] [degrees]
turn to heading [270] [degrees]

Python

Turning with Python:

drivetrain.turn_for(RIGHT, 90, DEGREES) drivetrain.turn_to_heading(270, DEGREES)

Practice Examples

Now let's combine everything you've learned! Try these examples in VEXcode VR to see how movement and turning work together.

Example 1: Drive and Turn Right

Make the robot drive forward, then turn right 90 degrees:

# Drive forward to the center drivetrain.drive_for(FORWARD, 800, MM) # Turn right to face a new direction drivetrain.turn_for(RIGHT, 90, DEGREES) # Drive forward in the new direction drivetrain.drive_for(FORWARD, 700, MM)

What it does: The robot drives straight, turns to face right, then drives in that new direction.

Example 2: Drive Forward and Return

Make the robot drive to a target and come back to the starting position:

# Set speed to maximum drivetrain.set_drive_velocity(100, PERCENT) # Drive forward drivetrain.drive_for(FORWARD, 800, MM) # Return to start drivetrain.drive_for(REVERSE, 800, MM)

What it does: The robot drives forward 800mm at maximum speed, then returns to its starting position.

Example 3: Navigate Using Headings

Use turn_to_heading() to ensure the robot faces exact directions:

# Set velocities drivetrain.set_drive_velocity(100, PERCENT) drivetrain.set_turn_velocity(100, PERCENT) # Drive forward drivetrain.drive_for(FORWARD, 800, MM) # Return to start drivetrain.drive_for(REVERSE, 800, MM) # Turn to face west (270 degrees) drivetrain.turn_to_heading(270, DEGREES) # Drive in the new direction drivetrain.drive_for(FORWARD, 700, MM)

What it does: The robot drives forward, returns, then uses a heading to face exactly west before driving in that direction.

Example 4: Draw an L-Shape

Create an L-shaped path by combining forward movement and turns:

# Set speeds drivetrain.set_drive_velocity(100, PERCENT) drivetrain.set_turn_velocity(100, PERCENT) # Drive forward (the vertical part of L) drivetrain.drive_for(FORWARD, 1550, MM) # Turn right 90 degrees drivetrain.turn_for(RIGHT, 90, DEGREES) # Drive forward (the horizontal part of L) drivetrain.drive_for(FORWARD, 700, MM)

What it does: The robot creates an L-shaped path on the playground.

Castle Crasher Challenge

Now it's time to put everything together! In this challenge, you'll use all the movement commands you've learned to navigate the Castle Crasher Playground and knock over all the castle buildings.

Challenge Goal: Program your robot to knock over all six castle buildings on the Castle Crasher Playground. The robot should drive to each castle, collide with it to knock it over, then move to the next one.

Understanding Sequences

A sequence is the specific order in which behaviors are performed. An action or event leads to the next ordered action in a sequence. Sequencing is critical because the VR Robot will only move exactly as the commands tell it to.

Important Concepts:
  • Order Matters: Commands execute from top to bottom, one at a time
  • Planning: Think about the path before you code
  • Testing: Run your project frequently to check if it works
  • Debugging: If something doesn't work, adjust the parameters and try again

Challenge Steps

  • Open VEXcode VR and create a new Python text project with the Castle Crasher Playground
  • Name your project "Unit2Challenge"
  • Study the playground to identify where all six castle buildings are located
  • Plan your route - decide which order to visit the castles
  • Set your drive and turn velocities (100% recommended for faster completion)
  • Add drive_for() and turn_for() or turn_to_heading() commands to navigate to each castle
  • Test your project by clicking "Start"
  • Debug and adjust distances/angles as needed
  • Keep testing until all castles are knocked over
  • Save your completed project!

Strategy Tips

Helpful Hints:
  • Start with the center castle first - it's straight ahead!
  • Drive through the center and continue to reach the castle on the opposite side
  • Use turn_to_heading() after collisions to ensure you're facing the right direction
  • You don't need to return to the start position
  • The castles don't need to be completely demolished - just displaced
  • Write comments in your code to remember your strategy
