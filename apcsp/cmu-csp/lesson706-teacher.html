Teacher Lesson Plan

Lesson 706: Unit 2 – Function Basics

Lesson Overview

Course: AP Computer Science Principles
Unit: Unit 2 – Functions, Mouse Events, and Conditionals
CMU Section: 2.1
Lesson Code: 706
Lesson Title: Function Basics
Class Length: 90 minutes

This lesson introduces one of the most important ideas in programming: functions. Students learn how to identify repeated code, convert it into reusable functions, and use parameters to make code flexible. They also practice debugging and begin independent CMU CS Academy exercises focused on writing and calling functions. The lesson supports long-term preparation for the AP CSP Create Performance Task by helping students write organized, modular, and reusable code.

Objectives

By the end of this lesson, students will be able to:

  • Explain what a function is and why programmers use functions.
  • Identify repeated code that can be replaced with a function.
  • Write basic Python functions using parameters.
  • Call functions with different arguments to produce different results.
  • Debug common function-related errors.
  • Apply functions within CMU CS Academy exercises.

Materials & Setup

  • Student computers
  • Internet access
  • CMU CS Academy student accounts
  • Projector for demonstrations
  • Student note-taking materials
  • Access to the CMU CS Academy "Function Basics" and "Debugging" lessons

Teacher Preparation:

  • Log into CMU CS Academy and open the Function Basics lesson
  • Prepare a short example program that repeats code so students can see how a function improves it
  • Verify all student accounts can log in before class

Teacher Moves (Scripted Talking Points + Demo Steps)

1. Opening Discussion

Say to students:

"Imagine you had to draw the same shape ten times in a program. Would it make sense to write the same code ten times?"

"Programmers use functions so they can write code once and reuse it many times."

Explain that functions help with:

  • Organization
  • Reusability
  • Debugging
  • Efficiency

2. Key Concepts to Emphasize

On the board or projector, introduce:

  • A function is a named block of code.
  • Functions may have parameters.
  • Functions must be defined before they are called.
  • Arguments are the values passed into parameters.

Example talking script:

"Think of a function like a recipe. The recipe has steps written once, but you can follow it with different ingredients each time."

3. Live Coding Demonstration

Show students a simple example:

Before functions (repeated code):

Rect(50, 100, 40, 60)
Rect(120, 100, 40, 60)
Rect(190, 100, 40, 60)

Ask:

  • "What is similar about these lines?"
  • "What is different?"

Convert to a function:

def drawBox(x):
    Rect(x, 100, 40, 60)

drawBox(50)
drawBox(120)
drawBox(190)

Highlight:

  • Parameters make the function flexible.
  • The body of the function is reused.

4. Transition to CMU Work

Explain:

"Now you will practice this same idea in the CMU CS Academy Function Basics lesson."

Student Tasks

In-Class Work

Students will:

  • Complete the Function Basics lesson in CMU CS Academy.
  • Work through the Debugging exercises related to functions.
  • Begin independent practice problems assigned in the CMU platform.

Expect students to:

  • Define functions with parameters.
  • Call functions multiple times with different arguments.
  • Fix syntax and logic errors involving functions.

Homework

Any unfinished CMU CS Academy exercises from today must be completed before the next class meeting.

Checks for Understanding (Formative)

Use the following strategies:

Ask students to explain in their own words:

  • "What is a parameter?"
  • "What is the difference between defining and calling a function?"

During independent work, ask:

  • "How could you turn this repeated code into a function?"
  • "What value is being passed into this parameter?"

Quick exit question:

  • "Name one reason functions make programs better."

Common Misconceptions & Fixes

Misconception Fix
Students think a function runs automatically Remind them functions must be called explicitly
Confusing parameters with arguments Use the phrase: "parameters are placeholders; arguments are actual values"
Forgetting to define functions before calling them Reinforce top-to-bottom execution order
Indentation errors in function bodies Review Python indentation rules

Differentiation & Accommodations

For Struggling Students

  • Provide a partially written function template.
  • Pair-program with a peer helper.
  • Focus on calling existing functions before writing new ones.

For Advanced Students

  • Challenge them to write functions with multiple parameters.
  • Ask them to refactor a larger program using several functions.

Assessment Notes

  • No formal grade today.
  • Progress will be monitored through CMU CS Academy completion data.
  • Mastery of functions is critical for later units and for the AP CSP Create Task.

Closure

"Today you learned how to make your programs cleaner and more powerful using functions."

Preview: "Next lesson we will build on this by adding mouse events and conditionals."

Ask: "How do functions help programmers avoid repeated work?"
