Teacher Lesson Plan

Lesson 707: Multiple Parameters + Functions Practice

Lesson Overview

Course: AP Computer Science Principles
Unit: Unit 2 – Functions, Mouse Events, and Conditionals
CMU Section: 2.1
Lesson Code: 707
Lesson Title: Multiple Parameters + Functions Practice
Class Length: 90 minutes

This lesson builds directly on the previous introduction to functions. Students extend their understanding by writing functions that use multiple parameters instead of a single input. Emphasis is placed on structured program design, reusability, and using parameters to control position, size, and other properties in graphics programs. The skills in this lesson directly support later AP CSP Create Performance Task work by helping students write modular, flexible code that can be reused in larger projects.

Objectives

By the end of this lesson, students will be able to:

  • Explain how functions can accept more than one parameter.
  • Write functions that use multiple parameters to control behavior.
  • Call functions with different sets of arguments.
  • Convert repeated code into structured multi-parameter functions.
  • Debug errors related to parameters and function calls.
  • Apply multi-parameter functions in CMU CS Academy practice exercises.

Materials & Setup

  • Student computers
  • Internet access
  • CMU CS Academy student accounts
  • Projector for teacher demonstrations
  • Student note-taking materials

Teacher Preparation:

  • Log into CMU CS Academy and open the parameter-based function exercises
  • Prepare a short example program that would benefit from multiple parameters
  • Confirm that all students can access the assigned CMU lessons

Teacher Moves (Scripted Talking Points + Demo Steps)

1. Warm-Up Review

Ask students:

"What problem do functions solve for programmers?"

"How did we use parameters in the last lesson?"

Guide students to recall:

  • Functions prevent repeated code.
  • Parameters let us change how a function behaves.

2. Introduce Multiple Parameters

Explain:

"A function does not have to be limited to one piece of information. Real programs often need several inputs to do useful work."

Show a simple transition:

Single parameter example:

def drawBox(x):
    Rect(x, 100, 40, 60)

Multiple parameter version:

def drawBox(x, y):
    Rect(x, y, 40, 60)

Point out:

  • Parameters are separated by commas.
  • Arguments must be provided in the same order.
  • More parameters mean more flexibility.

3. Demonstration Activity

Model a practical example:

Start with repeated code:

Rect(50, 80, 30, 30)
Rect(150, 120, 30, 30)
Rect(250, 200, 30, 30)

Convert to a structured function:

def drawSquare(x, y, size):
    Rect(x, y, size, size)

drawSquare(50, 80, 30)
drawSquare(150, 120, 30)
drawSquare(250, 200, 30)

Ask guiding questions:

  • "Which parts of the original code changed each time?"
  • "How do parameters help us control those parts?"

4. Connect to AP CSP Skills

Remind students:

  • Functions with parameters are a key requirement for the AP CSP Create Task.
  • Writing flexible, reusable code now will make their projects easier later.

Student Tasks

In-Class Work

Students will:

  • Complete the CMU CS Academy exercises focused on functions with multiple parameters.
  • Practice writing small drawing functions that accept several inputs.
  • Debug parameter-related errors in provided practice problems.

Teacher circulates to assist with:

  • Correct argument order
  • Proper function definitions
  • Debugging syntax and indentation

Homework

Any unfinished CMU CS Academy parameter exercises must be completed before the next class meeting.

Checks for Understanding (Formative)

Use quick informal checks such as:

  • "What is the difference between a parameter and an argument?"
  • "How many arguments does this function call need?"
  • "What will happen if the arguments are given in the wrong order?"

Observe students during practice:

  • Are they defining functions correctly?
  • Are they calling functions with the correct number of arguments?
  • Can they explain what each parameter controls?

Exit prompt:

  • "Explain in one sentence why multiple parameters make functions more useful."

Common Misconceptions & Fixes

Misconception Fix
Students mix up the order of arguments Emphasize that argument order must match parameter order
Forgetting commas between parameters Model proper function syntax repeatedly
Calling a function with too few arguments Review error messages and match calls to definitions
Trying to use variables that are not parameters Clarify scope and the purpose of parameters
Believing parameters keep values between calls Reinforce that parameters are temporary placeholders

Differentiation & Accommodations

For Struggling Students

  • Provide partially completed function templates.
  • Offer step-by-step checklists for writing functions.
  • Pair with a peer tutor for guided practice.

For Advanced Students

  • Encourage functions with four or more parameters.
  • Ask students to design a "drawHouse(x, y, size, color)" style function.
  • Challenge them to refactor earlier programs using multi-parameter functions.

Assessment Notes

  • No formal graded assessment today.
  • Progress measured through CMU CS Academy completion and accuracy.
  • Mastery of multi-parameter functions is essential for later units and AP Create Task success.

Closure

"Today you moved from simple functions to professional-style functions with multiple inputs."

Ask:

  • "How do multiple parameters make your programs more flexible?"
  • "Where might you use this idea in your own Create Task project?"

Preview: "Next, we will use functions together with events and conditionals to make interactive programs."
