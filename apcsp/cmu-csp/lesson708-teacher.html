Teacher Lesson Plan

Lesson 708: Mouse Events (onMousePress / onMouseRelease)

Lesson Overview

Course: AP Computer Science Principles
Unit: Unit 2 – Functions, Mouse Events, and Conditionals
CMU Section: 2.2
Lesson Code: 708
Lesson Title: Mouse Events (onMousePress / onMouseRelease)
Class Length: 90 minutes

This lesson introduces event-driven programming through mouse events. Students learn how to make programs interactive by responding to user clicks using onMousePress and onMouseRelease event handlers. They also learn to access mouse coordinates (mouseX and mouseY) to create dynamic, responsive programs. This is a critical shift from sequential programs to interactive programs and directly supports AP CSP Create Performance Task requirements for user input and program functionality.

Objectives

By the end of this lesson, students will be able to:

  • Explain what an event is in programming.
  • Define onMousePress and onMouseRelease event handler functions.
  • Use mouseX and mouseY to access click coordinates.
  • Create interactive programs that respond to user clicks.
  • Debug event-related errors in their programs.
  • Apply event handlers in CMU CS Academy exercises.

Materials & Setup

  • Student computers
  • Internet access
  • CMU CS Academy student accounts
  • Projector for teacher demonstrations
  • Student note-taking materials

Teacher Preparation:

  • Log into CMU CS Academy and open the mouse events lesson
  • Prepare a simple interactive example program to demonstrate
  • Test mouse event handlers to show common errors and fixes

Teacher Moves (Scripted Talking Points + Demo Steps)

1. Introduction to Events

Ask students:

"What makes a program interactive?"

"How does your phone know when you tap the screen?"

Explain:

"Up until now, our programs have run from top to bottom once. Today we're going to make programs that wait and respond when the user does something."

"An event is something that happens during a program—like a mouse click, a key press, or time passing."

"We write special functions called event handlers that only run when the event happens."

2. Introduce onMousePress

Core Concept:

"The onMousePress event handler runs every time the user clicks the mouse."

Show the basic structure:

def onMousePress(mouseX, mouseY):
    # This code runs when the user clicks
    Circle(mouseX, mouseY, 20, fill='red')

Point out:

  • The function name must be exactly onMousePress
  • It takes two parameters: mouseX and mouseY
  • These parameters tell you WHERE the user clicked
  • The code inside only runs when the user clicks

3. Demonstration: Build an Interactive Program

Live code a simple interactive drawing program:

Step 1 - Start with a background:

Rect(0, 0, 400, 400, fill='lightBlue')

Step 2 - Add an onMousePress handler:

def onMousePress(mouseX, mouseY):
    Circle(mouseX, mouseY, 30, fill='yellow')

Run the program and click multiple times to show circles appearing where you click.

Ask students:

  • "What happens when I click?"
  • "How does the program know where I clicked?"
  • "What would happen if I changed the circle's size or color?"

4. Introduce onMouseRelease

Explain the difference:

  • onMousePress – runs when you press the mouse button down
  • onMouseRelease – runs when you let the mouse button go
  • You can use both in the same program for different behaviors

Quick demo:

def onMousePress(mouseX, mouseY):
    Circle(mouseX, mouseY, 40, fill='blue')

def onMouseRelease(mouseX, mouseY):
    Rect(mouseX, mouseY, 30, 30, fill='green')

"Now different shapes appear when you press vs. when you release!"

5. Connect to AP CSP

Remind students:

  • The AP Create Task requires user input—mouse events are a perfect way to do this.
  • Interactive programs are more engaging and show deeper understanding.
  • You'll combine events with conditionals and functions to build complex projects.

Student Tasks

In-Class Work

Students will:

  • Complete the CMU CS Academy exercises on mouse events.
  • Practice writing onMousePress and onMouseRelease handlers.
  • Build interactive drawing programs using mouse coordinates.
  • Debug event handler errors (naming, indentation, parameters).

Teacher circulates to assist with:

  • Correct event handler function names (spelling matters!)
  • Proper use of mouseX and mouseY parameters
  • Understanding when code runs (setup vs. event-driven)

Homework

Any unfinished CMU CS Academy mouse event exercises must be completed before the next class meeting.

Checks for Understanding (Formative)

Use quick informal checks such as:

  • "What is the difference between onMousePress and onMouseRelease?"
  • "How do you access the location where the user clicked?"
  • "Why doesn't my event handler function run when the program starts?"

Observe students during practice:

  • Are they spelling event handler names correctly?
  • Are they using mouseX and mouseY properly?
  • Can they explain why their code only runs when they click?

Exit prompt:

  • "Describe one way you could use mouse events in a game or interactive project."

Common Misconceptions & Fixes

Misconception Fix
Misspelling event handler names (onMousepress, onmousePress, etc.) Emphasize exact spelling: onMousePress and onMouseRelease (capital M and P/R)
Calling the event handler function directly Explain that Python calls it automatically when the event happens
Forgetting to use mouseX and mouseY parameters Show examples where coordinates are needed vs. ignored
Expecting event code to run immediately when program starts Clarify setup code vs. event-driven code timing
Not understanding why shapes appear in different orders Review layering and the order in which code executes

Differentiation & Accommodations

For Struggling Students

  • Provide a template with the event handler structure already written.
  • Start with very simple tasks (e.g., "make a red circle appear when you click").
  • Pair with a partner for collaborative coding.

For Advanced Students

  • Challenge them to combine onMousePress and onMouseRelease for different effects.
  • Ask them to create a drawing tool with multiple colors or shapes.
  • Encourage them to use conditionals inside event handlers (preview of next lessons).

Assessment Notes

  • No formal graded assessment today.
  • Progress measured through CMU CS Academy completion and interaction quality.
  • Mastery of mouse events is essential for creating interactive AP Create Task projects.

Closure

"Today you transformed your programs from static to interactive. You can now respond to what the user does!"

Ask:

  • "How are event handlers different from regular functions?"
  • "What kinds of interactive programs could you build with mouse events?"

Preview: "Next, we'll add variables and properties so your programs can remember information and change over time."
