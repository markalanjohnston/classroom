AP Computer Science Principles

Internet Basics Review Quiz

ANSWER KEY — TEACHER USE ONLY
About this key: Student wording will vary. The answers below describe the concepts that should appear for full credit, drawn directly from the language used in the Internet Basics review lesson. Anything substantively equivalent counts. Watch for the listed misconceptions — those are the most common partial-credit traps.

1. Explain the difference between the Internet and the World Wide Web.

Look for both ideas:
  • Internet = the global network of networks — the underlying infrastructure of cables, routers, and standardized protocols that lets computers communicate.
  • World Wide Web = a service that runs on the Internet — a system of linked pages, programs, and files accessed via web browsers using HTTP/HTTPS.
Bonus credit if they note that other things also use the Internet without being the Web (email, streaming, gaming, messaging, file transfer).
Common partial-credit error: Treating the two as synonyms, or saying the Internet is software / Web is hardware. The key relationship is that the Web runs on top of the Internet.

2. What is the main job of DNS?

Look for: DNS translates human-readable domain names (like wikipedia.org) into IP addresses so that browsers and other software can locate the actual server.
Don't accept: "DNS delivers the webpage" or "DNS is what loads the site." DNS only does the lookup — HTTP/HTTPS and TCP/IP do the actual delivery.

3. When data is sent over the Internet, it is broken into smaller pieces called ______________________________.

Answer: packets

4. Name two pieces of information, or metadata, that are included in each packet.

Any two of:
  • Source address (sender's IP)
  • Destination address (receiver's IP)
  • Sequence number (so packets can be reassembled in order)
  • Other control / routing information
Don't accept: "A copy of the entire file" or "a copy of every other packet" — those are common distractors. Each packet only carries its own chunk plus the addressing/sequencing metadata.

5. Do packets always travel along the same path from sender to receiver? Explain.

Look for: No. Packets travel independently and may take different paths through the network. They can arrive out of order, and the receiving device reassembles them in the correct order using the sequence information in each packet.
Full credit needs: both the "no" and at least a brief explanation of independent routing / reassembly. Just "no" without explanation is partial credit.

6. What is fault tolerance, and how does the Internet achieve it?

Look for both parts:
  • Fault tolerance = the ability of a system to continue functioning when some of its components fail.
  • How: through redundancy (multiple/alternate paths between points) and dynamic routing (routers find new paths in real time when one fails or becomes congested), so traffic re-routes around the failure.
Don't accept: "Every device has backup power," "every packet is duplicated," or "one company monitors everything 24/7." Those are the distractor answers from the lesson.

7. Which protocol ensures that packets are delivered in the correct order and resends missing packets?

Answer: TCP (Transmission Control Protocol). Reliable delivery — reassembles packets in order and resends any that are lost.
Note: Accept "TCP/IP" as well; technically IP handles the routing and TCP handles the ordered/reliable part, but the pair is commonly named together.

8. What is the difference between HTTP and HTTPS?

Look for: HTTPS = HTTP + encryption (via TLS). HTTPS protects the data in transit between the browser and the server so that someone intercepting the connection can't read it. Plain HTTP sends data unencrypted.
Watch for the misconception: "HTTPS means the site is trustworthy / safe / not a scam." HTTPS only guarantees the connection is encrypted — not that the website is honest.

9. Define bandwidth and latency. You must define both.

Both definitions required:
  • Bandwidth = the maximum amount of data that can be transmitted per unit of time (capacity). Pipe-width analogy.
  • Latency = the delay between sending and receiving — how long one piece of data takes to arrive. Pipe-length analogy.
Common error: Conflating the two ("they're both about speed"). Award full credit only when each term has a distinct, accurate definition.

10. A video takes a long time to start playing, but once it starts, it plays smoothly in high quality. Is this a problem with bandwidth or latency? Explain.

Answer: Latency.

Reasoning to look for: Bandwidth must be fine because the video plays smoothly in high quality once it starts — that means lots of data is flowing. The delay before playback begins is a latency symptom: the round-trip / startup delay before any data starts arriving.
Full credit needs: both the correct identification (latency) and reasoning that ties the smooth playback to adequate bandwidth and the startup delay to latency. "Latency" alone is partial credit.
