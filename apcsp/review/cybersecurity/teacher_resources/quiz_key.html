AP Computer Science Principles

Cybersecurity & Encryption Review Quiz

ANSWER KEY — TEACHER USE ONLY
About this key: Student wording will vary. The answers below describe the concepts that should appear for full credit, drawn directly from the language used in the Cybersecurity & Encryption review lesson. Anything substantively equivalent counts. Watch for the listed misconceptions — those are the most common partial-credit traps and map directly to the AP MCQ wrong-answer choices.

1. Define encryption and decryption. You must define both.

Both definitions required:
  • Encryption = encoding data so that only authorized parties can read it.
  • Decryption = the reverse process — converting the encoded data back into its original, readable form.
Common error: Defining only one term, or describing encryption as "hiding" or "deleting" data. The key idea is encoding so only authorized parties (those with the correct key) can read it.

2. Explain the main difference between symmetric and asymmetric (public-key) encryption.

Look for both ideas:
  • Symmetric uses one shared key for both encryption and decryption. Both parties must already have the same secret key.
  • Asymmetric (public-key) uses a key pair: a public key (shared openly) for encryption and a private key (kept secret by the owner) for decryption.
Bonus credit if they note that symmetric has a key-distribution problem (you must somehow share the secret key first), and asymmetric solves it.
Common error: Saying asymmetric uses two keys without identifying which one encrypts and which one decrypts, or saying symmetric is "less secure" (it isn't — it's the key-sharing that's hard).

3. In public-key encryption, Alice wants to send Bob a confidential message. Whose key does Alice use to encrypt the message, and whose key does Bob use to decrypt it?

Answer:
  • Alice encrypts with Bob's public key.
  • Bob decrypts with Bob's private key.
The recipient's public key encrypts; only the matching private key (held only by the recipient) can decrypt.
Don't accept: "Alice's private key" (that's used for digital signatures, not for sending an encrypted message), or saying the public key can decrypt — it cannot. This is the single most-tested distinction in this unit.

4. What is the main advantage of public-key (asymmetric) encryption compared to symmetric encryption?

Look for: Two parties can communicate securely without first exchanging a secret key. Public-key encryption solves the key-distribution / key-sharing problem — you can publish your public key openly, and anyone can encrypt to you, even over an insecure channel.
Don't accept: "It's faster" (it's actually slower — that's why HTTPS uses public-key only to set up a symmetric session key) or "it can't be broken" (no system has that property).

5. What is the purpose of a digital certificate, and what kind of organization issues one?

Look for both parts:
  • Purpose: A digital certificate vouches that a particular public key really belongs to a particular website / server (i.e., it verifies the server's identity). Without it, an attacker could publish their own public key and impersonate the site.
  • Issuer: A trusted third party called a Certificate Authority (CA).
Common error: Saying the certificate "encrypts the data." It doesn't — it establishes identity. Encryption is performed by the keys; the certificate just confirms the public key is genuine.

6. A website is using HTTPS (the padlock icon is showing in the browser). Does this mean the website itself is trustworthy and safe? Explain.

Look for: No. HTTPS only guarantees that the connection between the browser and the server is encrypted (data in transit cannot be eavesdropped) and that the server's identity is verified by a certificate. It says nothing about whether the website is honest, accurate, or safe to use. A scam or phishing site can have a perfectly valid HTTPS certificate.
Full credit needs: both the "no" and a clear distinction between "encrypted connection" and "trustworthy website." Just "no" is partial credit.

7. What is Multi-Factor Authentication (MFA)? Name the three categories of authentication factors and give one example of each.

Look for the definition: MFA = an authentication method that requires at least two factors from different categories. (Two of the same factor is not MFA.)

Three categories with examples:
  • Something you KNOW — password, PIN, security question answer.
  • Something you HAVE — phone receiving a code, hardware security key (e.g., YubiKey), authenticator app.
  • Something you ARE — fingerprint, face scan, voice, iris (biometrics).
Full credit needs: all three categories named and a valid example for each. The "different categories" requirement should appear in the definition for full credit.

8. A website requires a password and the answer to a security question. Is this Multi-Factor Authentication? Explain why or why not.

Answer: No, this is not MFA.

Reasoning to look for: Both a password and the answer to a security question are in the same category — something you KNOW. MFA requires factors from different categories (e.g., know + have, know + are). Two pieces of "know" information is just multiple credentials, not multi-factor.
Full credit needs: both "no" and the explanation that both items are the same factor (something you KNOW). "No" without the why is partial credit.

9. Match each scenario to the correct threat term.

Answers:
  • a.Phishing — defining feature is social engineering: a fake message tricks the user into giving up credentials. The attack is on the person, not the system.
  • b.Keylogger — software (or hardware) that secretly records every keystroke, including passwords.
  • c.Rogue access point — an unauthorized Wi-Fi hotspot, often impersonating a legitimate network, used to intercept user traffic.
  • d.Worm — self-replicating malware that spreads autonomously across networks, no host file or user action required.
  • e.Virus — malware that attaches to a host file or program and spreads only when the user runs the host. Requires user action.
Common confusions: Students often swap virus and worm — the distinguishing rule is that a worm spreads on its own while a virus needs a host + user action. Malware alone is also a common wrong answer for any of d/e because it's the umbrella term — insist on the more specific word.

10. What is PII? Why is the combination of ZIP code, date of birth, and gender often considered a privacy risk?

Look for both parts:
  • PII (Personally Identifiable Information) = any information that can be used to identify, contact, or locate a specific individual — either on its own, or in combination with other information.
  • Why the combination is risky: Each piece alone is too generic to identify a specific person, but combining ZIP + date of birth + gender narrows the population enough to uniquely identify the majority of people. This is called aggregation — harmless-looking data becomes sensitive when combined.
Full credit needs: a definition of PII and the aggregation explanation. Bonus credit for using the word "aggregation" or referencing that ~87% of Americans can be identified this way (Latanya Sweeney's research, mentioned in the lesson).
