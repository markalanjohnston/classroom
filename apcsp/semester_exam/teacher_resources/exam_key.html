Part 1 — The Internet
1 — AThe Web is a service (linked pages via HTTP/HTTPS) that runs on the Internet. The Internet is the underlying infrastructure of networks and protocols. Internet vs. WWW
2 — CEmail (SMTP) uses the Internet but is not part of the Web. A, B, and D all describe Web activity (HTTP/HTTPS pages and hyperlinks). Internet vs. WWW
3 — CPacket switching: data is split into packets that travel independently, may take different paths, arrive out of order, and are reassembled at the destination. Packet switching
4 — AEach packet carries routing metadata (source, destination, sequence number) alongside its data chunk. The whole file is never duplicated into each packet. Packet switching
5 — DRedundant paths + dynamic routing = fault tolerance. If a router or cable fails, traffic reroutes. (C is also false: no single organization owns the Internet.) Fault tolerance
6 — BRedundancy = including extra components (like alternate paths) usable if a primary one fails. Fault tolerance
7 — CDNS (Domain Name System) translates a domain name into an IP address. It does not deliver the page itself. Protocols
8 — ATCP/IP: IP addresses and routes packets; TCP splits the data, ensures reliable in-order delivery, and reassembles it. Protocols
9 — BUDP is the faster, lighter protocol that does not wait to confirm delivery — ideal for live gaming/streaming where speed beats reliability. Protocols
10 — DHTTPS = HTTP + encryption (via TLS). It protects data in transit; it does not guarantee the site is honest. Protocols
11 — BIPv4's ~4.3 billion addresses are running out; IPv6's vastly larger address space lets the Internet keep growing. Encryption/speed are unrelated. IPv4 → IPv6
12 — COpen, nonproprietary standards mean any device from any manufacturer can interoperate — enabling scalability. Open standards
13 — B"How long until I get a response" = latency. Bandwidth is fine (video plays in high quality once started); the startup delay is the round-trip lag. Bandwidth vs. latency
14 — DThe Internet is a network of networks with no single owner, held together by shared open protocols. A, B, and C are the classic misconceptions. Internet basics
Part 2 — Cybersecurity & Encryption
15 — BEncryption = encoding data so only authorized parties (those with the key) can read it. It is not deletion, backup, or compression. Encryption
16 — ASymmetric encryption uses one shared key for both encrypting and decrypting. (A public/private key pair is asymmetric.) Encryption
17 — DAlice encrypts with Bob's public key; only Bob's matching private key can decrypt. The public key cannot decrypt. (Most-tested fact in this unit.) Public-key
18 — CPublic-key encryption solves the key-distribution problem: parties can communicate securely without first sharing a secret. (It is actually slower, not faster.) Public-key
19 — AA digital certificate vouches that a public key truly belongs to a specific site — it verifies identity. It does not itself encrypt the data. Certificates
20 — BCertificates are issued by a trusted third party, a Certificate Authority (CA). Certificates
21 — DHTTPS guarantees the connection is encrypted in transit (and the server's identity via the certificate). It does not guarantee the site is honest, unhackable, or anonymous. A phishing site can have valid HTTPS. HTTPS
22 — CA lockout delay is not an authentication factor. A, B, D each combine two different categories (know/have/are). A password + a delay timer is just one factor plus a timer. MFA
23 — BPassword = something you know; phone receiving the code = something you have. MFA
24 — DA rogue access point is an unauthorized Wi-Fi hotspot impersonating a real network to intercept traffic. No software runs on the victim's device (not a virus/keylogger) and no fake message is sent (not phishing). Threats
25 — APhishing — social engineering that tricks the user into handing over credentials via a fake message/site. No malware required. Threats
26 — AA worm self-replicates and spreads autonomously across networks; a virus attaches to a host file and needs user action to spread. (Both are types of malware.) Threats
27 — CZIP code + date of birth + gender uniquely identifies the large majority of people — the canonical "aggregation makes data sensitive" example. PII
Part 3 — Reading Pseudocode
28 — B← is assignment: evaluate the right side and store it in the variable on the left. Assignment
29 — C= is the equality test — it produces true or false and changes no variable. Equality
30 — DMOD is the remainder: 17 = 3×5 + 2, so 17 MOD 5 is 2. (3.4 is the quotient 17 / 5.) MOD
31 — ARANDOM(a, b) returns a random integer from a to b inclusive, so 1, 2, 3, or 4. RANDOM
32 — A"Inclusive" range needs ≥ and ≤ joined by AND: (x ≥ 50) AND (x ≤ 80). B is impossible; C is always true; D excludes the endpoints. Boolean
33 — BOR is true when at least one operand is true, including when both are true (it is not exclusive-or). Boolean
34 — CAn IF / ELSE always runs exactly one block; two separate IFs independently can both run or both be skipped. Selection
35 — AREPEAT UNTIL stops when the condition becomes true. ("Repeat until true," not "while true.") Iteration
36 — DAP CSP lists are 1-indexed, so aList[1] is the first element, 10. (aList[0] would be an error.) Lists
37 — CBecause lists are 1-indexed, the last index equals the length, so aList[LENGTH(aList)] is the last element. Lists
38 — BFOR EACH assigns the values (10, 20, 30), not the indices. Iteration
39 — DTrace it out: each loop does x ← x + y then y ← y - 1. Final x is 15.
