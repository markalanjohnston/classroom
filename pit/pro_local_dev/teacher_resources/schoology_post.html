PIT Practicum: Transitioning to Professional Local Development

Lesson Goal: Move one of your CMU Academy programs out of the browser and into a real professional local development environment — VS Code, Python, CMU Graphics, and Git — the same toolchain professional developers use every day.

All detailed instructions, commands, checkpoints, and the full grading rubric live on the class website. This Schoology page is intentionally brief — the website is your lab manual for this practicum.

Learning Objectives

By the end of this 3-hour practicum, you will be able to:

  1. Verify a working professional Python environment (VS Code + Python) on a local machine.
  2. Install cmu-graphics with pip and run a CMU Graphics program locally.
  3. Migrate one of your CMU Academy programs into VS Code with improved formatting, naming, and comments.
  4. Initialize a local Git repository and create meaningful commits.
  5. Articulate why version control matters — and recognize parallel “workflow” and “versioning” practices in other industries.
  6. Submit a professional PDF deliverable documenting your work.

Why This Matters Beyond CS

Even if you never write code professionally, in this practicum you’re practicing a bigger skill: learning a new workflow quickly and following it without skipping steps — the same skill used by pilots running pre-flight checklists, accountants closing the books, and surgeons running a sterile-field protocol. The lesson includes “Profession Connection” callouts showing how the ideas of workflow, package management, documentation, version control, and iteration appear in law, medicine, journalism, finance, agriculture, sports, and more.
