What You Will Do (High Level)

  1. Phase 1 — Deployment & Configuration: Install VS Code and Python on your assigned machine. Verify PATH, terminal access, and run a Python program from both the terminal and from VS Code.
  2. Phase 2 — Capability Testing: Validate which Python capabilities the system actually supports — pip, CMU Graphics, tkinter, turtle, image/sound assets, internet access, and local file I/O.
  3. Phase 3 — Boundary Testing: Professionally probe the limits of the managed environment. Compare findings across machines.
  4. Phase 4 — Capability Report: Produce a professional PDF summarizing what works, what is restricted, and whether this lab is suitable for local Python development.

Format

  • 3-hour block, work independently or in small troubleshooting groups (2–3).
  • Mr. Johnston is acting as a roaming support technician, not delivering direct instruction.
  • Pace yourself. The website includes a suggested time breakdown.
  • Escalate blockers only after attempting at least one documented workaround — bring the exact error text with you.

Submission

Deliverable: A single PDF named LASTNAME_FIRSTNAME_PythonCapabilityReport.pdf, attached to this Schoology assignment before the end of the block.

The report must include the ten sections listed on the website (cover page, executive summary, system inventory, deployment findings, capability test matrix, boundary findings, identified failures/restrictions, recommendations, overall verdict, appendix).

Grading

Your report is evaluated as professional documentation, on five criteria:

  • Completeness — all required sections, every test row filled in
  • Evidence — screenshots and exact error text
  • Accuracy — claims match the captured evidence
  • Professional communication — tone, clarity, formatting, proofread
  • Insight — recommendations and verdict grounded in your data

Honest FAIL or RESTRICTED results, well documented, are worth full credit. Inflating results or copying a classmate's findings is academic dishonesty.
