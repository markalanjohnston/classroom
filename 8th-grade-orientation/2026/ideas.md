# 8th Grade Orientation Ideas

I have 8th grade students coming tomorrow morning. My classroom will be empty. I will have a couple seniors assisting me. I need to be able to show students that come into my room what they might experience here.

I have a bunch of Arduinos that I could have setup but I don't have any components setup. I could connect a couple of LEDs. I could then have a couple computers logged into Tinkercad Codeblocks, and a couple with the Arduinos with the Arduino IDE. I could have some Codecademy programs running on a few computers.

I am just used to having more physical items like I did when I taught Robotics/Engineering. I DO want to use some robotics next year for programming — should I bring a robot from home and setup a station with VEXcode and a robot they can program and run?

I would also like to have a flyer about Fundamentals of Computer Science (they can take it Freshman year — P-TECH students get first dibs but other students can join too). Any other ideas about what I could do to make the 8th grade orientation better?

---

Yes, bring the robot from home if it is not a major hassle. For 8th graders, moving parts beat screens almost every time. Even if the class is mostly programming, a robot makes the programming feel real.

Set the room up as a **"Computer Science is not just typing code"** showcase with several quick stations.

---

## Best Setup for Tomorrow

### 1. Robot Programming Station

Use this as the attention-grabber.

**Setup:**
- VEX robot or similar robot from home
- VEXcode open on a computer
- Tape a simple square or maze on the floor
- Seniors help students change one number in the code and rerun it

**Student experience:**

> "Change the distance. Change the turn. Run it again."

This shows programming as problem-solving, testing, debugging, and controlling real hardware.

**Why it works:** They immediately see that code can control physical things.

---

### 2. Arduino LED Station

This is worth doing even if it is simple.

**Setup:**
- 2–3 Arduinos
- LEDs connected
- Arduino IDE open on one or two computers
- Tinkercad Circuits open on a couple more

**Simple activity:**
- Change the blink speed
- Make two LEDs alternate
- Modify comments or numbers in the code

**Example prompt on the screen:**
```cpp
// Change these numbers and see what happens
digitalWrite(13, HIGH);
delay(1000);
digitalWrite(13, LOW);
delay(1000);
```

Students do not need to understand all of it. They just need to see:

> "I changed the code, and the device changed behavior."

---

### 3. Tinkercad Virtual Circuit Station

This is good for students who are nervous about touching hardware.

**Setup:**
- Tinkercad Circuits already open
- LED blink simulation loaded
- Code view open

**Student task:**

> "Can you make the light blink faster?"

This also lets you say:

> "Even when we do not have physical parts, we can simulate and test ideas."

---

### 4. Codecademy / Coding Puzzle Station

Use this as a quieter station, but do not make it the centerpiece.

**Setup ideas:**
- Python basics
- JavaScript basics
- Simple "change the message" or "fix the code" activity

**Better than a full lesson:** Have something already running where they can make one visible change.

**Examples:**
- Change text on a webpage
- Change a color
- Fix a broken print statement
- Make a character move

Avoid anything that requires creating accounts.

---

### 5. "What You'll Build in High School CS" Display

This could just be projected or shown on a few screens.

**Show examples of:**
- Games
- Apps
- Robots
- Circuits
- Websites
- AI-assisted coding
- Cybersecurity concepts
- AP Computer Science pathway
- Industry connections

Even if you do not have physical artifacts, screenshots of student projects or demo projects can help.

---

## Room Layout Suggestion

Have signs at each station:

| Station | Name | What students do |
|---------|------|-----------------|
| 1 | Program a Robot | Change code and watch the robot move |
| 2 | Arduino Lights | Change code to control LEDs |
| 3 | Virtual Circuits | Build/test circuits on screen |
| 4 | Coding Challenge | Modify a small program |
| 5 | CS Pathway | See what classes you can take next |

Put the robot station closest to the entrance or most visible from the door.

---

## What Seniors Should Do

Give each senior a specific role.

**Senior 1: Robot Station**
They help students run the robot and say:

> "This is what debugging looks like. We try something, test it, and adjust."

**Senior 2: Arduino / Tinkercad Station**
They help students change numbers in code and explain:

> "You do not have to know everything yet. The class teaches you step by step."

If you have extra seniors, have one act as a greeter:

> "Have you coded before? No? That's fine. Most students start from scratch."

That last point matters. A lot of 8th graders will assume computer science is only for students who already know how to code.

---

## Message You Want Students to Hear

Repeat this all morning:

> "You do not need to already know how to code. Fundamentals of Computer Science is where you start."

Also:

> "Computer science is not just for people who want to be programmers. It helps with engineering, health care, business, cybersecurity, game design, robotics, science, and almost every modern career."

---

## Flyer Content

**Title:** Fundamentals of Computer Science

**Subtitle:** Start here if you are interested in coding, technology, robotics, games, cybersecurity, or engineering.

**What you will learn:**
- How to write your first real programs
- How computers follow instructions
- How to solve problems using code
- How to create interactive projects
- How coding connects to robotics, apps, games, and real careers
- How to prepare for future AP Computer Science classes

**You do not need experience**

No coding experience is required. This class is designed as a starting point for students who are curious and willing to try.

**Who can take it?**
- Freshmen can take Fundamentals of Computer Science
- P-TECH students get first priority
- Other students may be able to join if space is available

**Why take it?**

Computer science can help you in:
- Engineering
- Robotics
- Cybersecurity
- Game design
- App and web development
- Business
- Health care technology
- Data analysis
- Artificial intelligence
- Military and aerospace careers

**One-line pitch:** If you like solving problems, building things, or figuring out how technology works, this is a good class to start with.

## Extra Quick Ideas

### Have a "Change One Thing" Rule

At every station, students should be able to change one thing and see a result.

**Examples:**
- Change robot distance
- Change LED blink speed
- Change text on a webpage
- Change a color
- Change a variable
- Fix one broken line of code

That is better than trying to explain too much.

### Put "You Belong Here" Somewhere Visible

Something like:

> Never coded before? Good. This class is built for beginners.

That directly addresses the intimidation factor.

### Use a Simple Pathway Poster

```
9th Grade:  Fundamentals of Computer Science
10th Grade: AP Computer Science Principles
11th Grade: Computer Science 1 / Java
12th Grade: AP Computer Science A
```

> Start with Fundamentals. Build toward AP credit, industry certifications, and real programming skills.

### Let Seniors Speak Briefly

Ask them to answer:
- What surprised you about computer science?
- What was hard at first?
- What helped you improve?
- Why should freshmen try it?

8th graders may trust students more than adults.

---

## Best Version if You Have Limited Prep Time

Do these four things:

1. Bring the robot.
2. Set up one Arduino LED blink station.
3. Set up two Tinkercad/code stations where students change one number.
4. Print a simple Fundamentals of Computer Science flyer.

That is enough to make the room feel active, hands-on, and approachable.
